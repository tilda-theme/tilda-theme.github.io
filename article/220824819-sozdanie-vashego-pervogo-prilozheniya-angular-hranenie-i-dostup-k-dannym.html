Создание вашего первого приложения Angular: Хранение и доступ к данным...

Code Angular
Tutorial

В первом уроке этой серии мы узнали, с чего начать создание приложения Angular. После успешного завершения этого урока, теперь вы должны иметь ваше первое рабочее приложение Angular "Интересные факты о странах". Прежде чем создавать компоненты, которые могут быть отображены на экране, мы создадим некоторые классы и определим некоторые функции, которые сделают эти компоненты полезными.

В этом уроке мы сосредоточимся на создании класса Country, который будет содержать различные свойства, значение которых мы хотим отображать для пользователя. Затем мы создадим еще один файл country-data.ts. Этот файл будет содержать информацию о всех странах в нашем приложении. Наш третий файл будет называться country.service.ts. Название может звучать причудливо, но файл просто будет содержать класс CountryService со всеми необходимыми функциональными возможностями для извлечения и сортировки информации, представленной файлом country-data.ts.

Создание класса Country

В папке src/app вашего приложения Angular создайте файл country.ts. Добавьте в него следующий код.

export class Country {    name: string;    capital: string;    area: number;    population: number;    currency: string;    gdp: number;}

Приведенный выше код TypeScript определяет класс Сountry с шестью различными свойствами для хранения информации о разных странах. Название страны, ее столица и ее валюта хранятся в виде строки. Однако его территория, население и ВВП хранятся как число. Мы будет импортировать класс Country во многих местах, поэтому я добавил ключевое слово export перед определением класса.

Создание массива стран

Следующий шаг включает в себя создание файла country-data.ts для хранения сведений обо всех странах в виде массива объектов Country. Мы будем импортировать класс Country в этом файле и затем экспортировать константу const COUNTRIES, которая хранит массив объектов стран.

Вот код country-data.ts. Так же, как и country.ts необходимо создать этот файл в папке src/app.

import { Country } from "./country";export const COUNTRIES: Country[] = [  {    name: "Russia",    capital: "Moscow",    area: 17098246,    population: 144463451,    currency: "Russian Ruble",    gdp: 1283162  },  {    name: "Canada",    capital: "Ottawa",    area: 9984670,    population: 35151728,    currency: "Canadian Dollar",    gdp: 159760  },  {    name: "China",    capital: "Beijing",    area: 9596961,    population: 1403500365,    currency: "Renminbi (Yuan)",    gdp: 11199145  },  {    name: "United States",    capital: "Washington, D.C.",    area: 9525067,    population: 325365189,    currency: "United States Dollar",    gdp: 18569100  },  {    name: "Japan",    capital: "Tokyo",    area: 377972,    population: 12676200,    currency: "Yen",    gdp: 4939384  }];

Первая строка в этом файле импортирует класс Country из файла country.ts, расположенного в том же каталоге. Если вы удалите эту строку из файла, TypeScript выдаст вам следующую ошибку:

Cannot find name "Country".

Без оператора import TypeScript не будет знать, что означает массив типа Country. Поэтому убедитесь, что вы импортировали правильный класс и указали расположение country.ts правильно.

После импорта класса Country, мы идем дальше и создаем массив объектов Country. Мы будет импортировать этот массив стран для использования в других файлах, поэтому мы добавим ключевое слово export в этот массив. В настоящее время есть пять различных объектов Country в массиве. Каждый из этих пяти объектов содержит пары ключ-значение, которые содержат имя свойства и его значение для определенного объекта или страны.

Если вы попытаетесь добавить дополнительное свойство в массив, которое не было определено внутри класса Country, вы получите следующее сообщение об ошибке:

Object literal may only specify known properties, and "president" does not exist in type "Country"

В этом случае я пытался сохранить имя президента как строку (string) внутри свойства с именем president. Поскольку такое свойство не было определено, мы получили сообщение об ошибке. Иногда может потребоваться задать свойство только для определенных объектов, а не для других. В таких случаях можно пометить свойство как необязательное внутри определения класса. Я обсуждал это более подробно в уроке, который охватывает тему интерфейсов TypeScript.

Пока просто убедитесь, что имена всех свойств совпадают с именами внутри определения класса. Кроме того, убедитесь, что значение каждого свойства имеет тот же тип, который объявлен в определении класса.

Создание класса CountryService

После создания нашего класса Country и массива COUNTRIES, мы можем теперь наконец написать несколько функций для обработки данных стран. Нам нужно будет импортировать и класс Country, и массив COUNTRIES внутри файла нашей службы. В этом файле нужно будет импортировать массив COUNTRIES для того, чтобы иметь доступ к данным. Также в файле придется импортировать и класс Country для того, чтобы иметь возможность работать с данными из массива COUNTRIES.

Мы также импортируем другие зависимости, такие как Injectable из ядра Angular для того, чтобы наш класс CountryService стал доступен для загрузчика для вставки в другие компоненты.

По мере роста вашего приложения, различные модули должны иметь возможность общаться друг с другом. Предположим, что ModuleA требует ModuleB для правильной работы. В таких случаях мы бы назвали ModuleB зависимостью ModuleA.

Простой импорт модуля, который нам нужен, в другой файл работает большую часть времени. Однако иногда мы должны решить, создавать ли единственный экземпляр классов ModuleB, который будет использоваться всем приложением, или же мы должны создавать новый экземпляр каждый раз, когда используется модуль. В нашем случае мы будет вставлять один экземпляр нашего класса CountryService для всего приложения.

Вот код country.service.ts:

import { Injectable } from "@angular/core";import { Country } from "./country";import { COUNTRIES } from "./country-data";@Injectable()export class CountryService {  constructor() { }  getCountries(): Country[] {    return COUNTRIES;  }  getPopulatedCountries(): Country[] {    return COUNTRIES.sort((a, b) => b.population - a.population).slice(0, 3);  }  getLargestCountries(): Country[] {    return COUNTRIES.sort((a, b) => b.area - a.area).slice(0, 3);  }  getGDPCountries(): Country[] {    return COUNTRIES.sort((a, b) => b.gdp - a.gdp).slice(0, 3);  }  getCountry(name: string): Country {    return COUNTRIES.find(country => country.name === name);  }}

Декоратор @injectable используется для определения служебного класса, который может потребовать внедрение зависимостей. Однако, добавление @injectable к служебным классам является обязательной нормой программирования, поэтому мы делаем это в любом случае.

После этого мы пишем различные методы для класса, которые принимают массив COUNTRIES и либо возвращают его непосредственно, либо с помощью определенных критериев сортируют его и затем возвращают часть массива.

Ожидается, что метод getCountries() возвращает все объекты Country, и поэтому он возвращает весь массив COUNTRIES без каких-либо изменений.

GetPopulatedCountries() принимает массив COUNTRIES и сортирует его в порядке убывания по населению различных стран. Затем мы используем метод Array.slice() для возврата первых трех стран (с индексами 0, 1 и 2) из массива. Методы getLargestCountries() и getGDPCountries() работают аналогичным образом.

Метод getCountry() принимает название в качестве своего аргумента и возвращает объект страны, чье свойство name имеет то же значение, что и переданное в качестве аргумента название.

Добавление CountryService в app.module.ts

Сервис, который вы создаете - это просто класс в Angular до тех пор, пока вы не зарегистрировали его с помощью загрузчика зависимостей Angular. Загрузчик Angular один отвечает за создание экземпляров служб и внедрение их в различные классы, которые требуют этих служб. Нам нужно зарегистрировать службу с помощью оператора, прежде чем загрузчик сможет создать эту службу.

Существует два распространенных способа зарегистрировать любую службу: с помощью оператора @Component или оператора @NgModule. Использование оператора @Component имеет смысл, когда вы хотите ограничить доступ службе только для определенного компонента и всех его вложенных компонентов. Использование оператора @NgModule имеет смысл, когда требуется, чтобы множество компонентов имели доступ к этой службе.

В нашем случае мы будем использовать CountryService с множеством компонентов нашего приложения. Это означает, что мы должны зарегистрировать его с помощью оператора @NgModule один раз, вместо отдельной его регистрации оператором @Component для каждого компонента.

В настоящее время ваш файл app.module.ts должен выглядеть следующим образом:

import { BrowserModule } from "@angular/platform-browser";import { NgModule } from "@angular/core";import { AppComponent } from "./app.component";@NgModule({  declarations: [    AppComponent  ],  imports: [    BrowserModule  ],  providers: [],  bootstrap: [AppComponent]})export class AppModule { }

Добавьте import в файл app.module.ts и добавьте службу в массив оператора @NgModule. После внесения этих изменений, файл app.module.ts должен выглядеть следующим образом:

import { BrowserModule } from "@angular/platform-browser";import { NgModule } from "@angular/core";import { AppComponent } from "./app.component";import { CountryService } from "./country.service";@NgModule({  declarations: [    AppComponent  ],  imports: [    BrowserModule  ],  providers: [CountryService],  bootstrap: [AppComponent]})export class AppModule { }

Класс CountryService теперь будет доступен для всех компонентов, которые мы создаем для нашего приложения.

Заключительные соображения

Второй урок данной серии завершается успешным созданием трех файлов: country.ts, country-data.ts и country.service.ts.

Файл country.ts используется для создания класса Country с различными свойствами, такими как название, валюта, население, площадь и т.д. Файл country-data.ts используется для хранения массива объектов страны, которые имеют информацию о разных странах. Файл country.service.ts содержит класс службы с различными методами для доступа к данным страны из массива COUNTRIES. То, что мы написали все эти методы отдельно внутри класса службы, позволяет нам получить к ним доступ внутри других компонентов приложения из центрального расположения.

В последнем разделе мы зарегистрировали наш сервис с помощью оператора @NgModule, чтобы сделать его доступным для использования внутри различных компонентов.
