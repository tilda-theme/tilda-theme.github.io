Манипулирование- HTML5 canvas с использованием Konva: часть 5, События...

Tutorial

Если вы следили за этой серией с самого начала, то теперь вам должно быть очень комфортно работать с фигурами, группами и слоями. Вы также можете легко рисовать некоторые основные и сложные фигуры на холсте (canvas) с помощью Konva. Если вы планируете использовать Konva для создания интерактивных приложений или игр, следующим логическим шагом будет изучение привязки событий к различным фигурам на сцене (stage).

Из этого руководства вы узнаете, как привязать события к любой фигуре с помощью Konva. Вы также узнаете о делегировании и всплытии событий. Иногда вам может понадобиться управлять областью фигуры, а также программно запускать события. Далее мы будем обсуждать эти две темы.

Привязка событий к фигуре

С помощью метода on() можно привязать различные события к любой фигуре, созданной с помощью Konva. Все, что вам нужно сделать, это передать имя события в качестве первого параметра, и имя функции которая будет выполняться, когда это событие происходит в качестве второго параметра. Вы можете использовать Konva для обнаружения событий mouseup, mousedown, mouseenter, mouseleave, mouseover, mousemove, click, и dblclick. Кроме того, Konva позволяет обнаружить события wheel, dragstart, dragmove, и dragend.

Вот пример, который обнаруживает события mousedown и mouseleave на правильном многоугольнике (шестиугольник). Аналогично, меньший круг привязан к событиям mouseover и mouseup, а больший круг привязан к событиям mouseenter, mouseleave и mousemove.

var canvasWidth = 600;var canvasHeight = 400;var stage = new Konva.Stage({  container: "example",  width: canvasWidth,  height: canvasHeight});var layerA = new Konva.Layer();var polyA = new Konva.RegularPolygon({  x: 125,  y: 125,  sides: 6,  radius: 80,  fill: "yellow",  stroke: "black",  strokeWidth: 5});var circA = new Konva.Circle({  x: 275,  y: 225,  height: 100,  fill: "orange",  stroke: "black"});var circB = new Konva.Circle({  x: 475,  y: 275,  radius: 100,  fill: "red",  stroke: "black"});layerA.add(polyA, circA, circB);stage.add(layerA);polyA.on("mousedown", function() {  polyA.sides(polyA.sides() + 1);  layerA.draw();});polyA.on("mouseleave", function() {  var totalSides = polyA.sides();  if(totalSides > 3) {    polyA.sides(polyA.sides() - 1);  }  layerA.draw();});circA.on("mouseover", function() {  circA.strokeWidth(10);  layerA.draw();});circA.on("mouseup", function() {  circA.strokeWidth(5);  layerA.draw();});circB.on("mouseenter", function() {  stage.container().style.cursor = "crosshair";});circB.on("mouseleave", function() {  stage.container().style.cursor = "default";});circB.on("mousemove", function() {  var pointerPos = stage.getPointerPosition();  var r = pointerPos.x % 255;  var g = pointerPos.y % 255;  circB.fill("rgb(" + r + ", " + g + ", 100)");  layerA.draw();});

Если пользователь нажимает любую кнопку мыши, когда курсор находится внутри обычного многоугольника, мы увеличиваем количество сторон многоугольника на 1. Метод sides() может использоваться без параметра, чтобы получить количество сторон многоугольника или использовать с одним параметром, чтобы установить количество сторон многоугольника. Вы также можете получить количество сторон, используя getSides(), и установить количество сторон с помощью setSides(). Стороны многоугольника уменьшаются на единицу, когда курсор мыши покидает многоугольник.

Для меньшего круга событие mouseover используется для установки значения ширины обводки в 10. Событие mouseup изменяет значение ширины обводки на 5. Помните, что событие mouseup должно происходить внутри самого круга. Например, ширина обводки не изменится на 5, если вы нажмете кнопку мыши внутри круга, а затем отпустите ее только после того, как курсор находится за пределами круга.

В случае большего круга мы используем событие mousemove, чтобы изменить его цвет заливки fill. Мы также меняем курсор большого круга, используя stage.container().style.cursor всякий раз, когда курсор перемещается в круг и из него.

Одна важная вещь, которую вы должны иметь в виду, состоит в том, что вам нужно вызвать метод draw() на соответствующем слое, если слушатели событий для любой фигуры привели к изменению атрибутов, таких как цвет заливки, ширина обводки и т.д. В противном случае изменения не будет отражено на холсте.

Вы не обязаны связывать одно событие за один раз с фигурой. Вы можете передать строку с разделителями, содержащую несколько типов событий, методу on(). Это свяжет все события, перечисленные в строке, с этой конкретной фигурой.

Konva также поддерживает соответствующие мобильные версии всех этих событий. Например, вы можете зарегистрировать touchstart, touchmove, touchhend, tap, dbltap, dragstart, dragmove и dragend с помощью Konva на мобильных устройствах.

Вы также можете запустить любое из этих событий для определенной фигуры или фигур с помощью метода fire(). Аналогично, Konva позволяет запускать пользовательские события, такие как throwStones.

Удаление слушателей событий

Вы можете удалить слушателей событий, прикрепленных к фигуре, с помощью Konva метода off(). Вам просто нужно указать имя события, которое вы не хотите слушать.

Вы можете также создать несколько привязок событий для одной фигуры. Например, предположим, что у вас есть круг, и вы хотите увеличить радиус круга каждый раз, когда курсор мыши перейдет через него, до определенного предела. Вы также можете изменить цвет заливки круга в каждом событии mouseover.

Один из вариантов состоит в том, чтобы выполнить обе эти задачи внутри одного приемника событий mouseover и прекратить обновление радиуса позже. Другим вариантом является создание двух слушателей событий mouseover с разными пространствами имен для их идентификации. Таким образом, вы сможете увеличить радиус и изменить цвет заливки независимо.

circA.on("mouseover.radius", function() {  var curRadius = circA.radius();  if(curRadius < 150) {    circA.radius(curRadius + 5);    layerA.draw();  } else {    circA.off("mouseover.radius");  }});circA.on("mouseover.fillcolor", function() {  var h = Math.floor(Math.random()*360);  var color = "hsl(" + h + ", 60%, 60%)";  circA.fill(color);  layerA.draw();});

Вы должны заметить, что я добавил layerA.draw() внутри обоих слушателей. Если вы не сможете добавить его в приемник mouseover.fillcolor, цвет перестанет обновляться, как только радиус станет 150.

Вместо того, чтобы удалять одного слушателя событий за раз, вы также можете прекратить прослушивание всех событий, связанных с формой, с помощью метода setListening(). Вы можете передать true и false этому методу, чтобы включить on и выключить off слушателей событий. Имейте в виду, что вам также придется перерисовать хит-график затронутого слоя, вызвав метод drawHit() сразу после вызова setListening().

Делегирование и всплытие событий

Вместо привязки событий непосредственно ко всем фигурам, присутствующим на слое, вы также можете привязать событие к самому слою. После этого вы можете определить, какая фигура вызвала событие, используя свойство target объекта события. Таким образом, Konva позволяет эффективно делегировать события от родителя к своим дочерним элементам.

Предположим, вы слушаете события клика на круге, нарисованном на слое в Konva. Одно и то же событие клика срабатывает и на группу контейнеров, а также на слой-контейнер. Это может быть или не быть предполагаемым поведением. Если вы хотите, чтобы событие не всплывало от фигуры к слою-контейнеру, вы можете установить свойству cancelBubble для объекта события значение true.

var canvasWidth = 600;var canvasHeight = 400;var stage = new Konva.Stage({  container: "example",  width: canvasWidth,  height: canvasHeight});var layerA = new Konva.Layer();var circA = new Konva.Circle({  x: 300,  y: 200,  height: 100,  fill: "orange",  stroke: "black",  name: "Orange Circle"});var starA = new Konva.Star({  x: 125,  y: 125,  innerRadius: 25,  outerRadius: 75,  rotation: 90,  fill: "blue",  stroke: "black",  name: "Blue Star"});var ringA = new Konva.Ring({  x: 475,  y: 275,  innerRadius: 25,  outerRadius: 75,  fill: "brown",  stroke: "black",  name: "Brown Ring"});var textA = new Konva.Text({  text: "",  fontFamily: "Calibri",  fontSize: 24,  fill: "black",  x: 10,  y: 10});layerA.add(circA, starA, ringA, textA);stage.add(layerA);layerA.on("click", function(e) {  var shapeName = e.target.attrs.name;  textA.setText(shapeName);  layerA.draw();});

Я использовал свойство name для присвоения имени каждой из наших форм. Метод setText() затем используется для изменения текста внутри textA на имя формы, которую мы только что нажали.

Пользовательские области попадания

В приведенном выше примере кольцо зарегистрировало клик на нем, когда щелчок произошел между внутренним и внешним кругом. Что делать, если вы хотите зарегистрировать клик внутри меньшего круга? Konva позволяет определить пользовательские области попадания, используя свойство hitFunc. Это свойство принимает функцию как ее значение, и эта функция используется для создания пользовательской области попадания.

В следующем примере показано, как создавать пользовательские области попадания. Теперь вы можете щелкнуть по области между звездочками и все еще зарегистрировать клик. С помощью пользовательских областей попадания вы можете убедиться, что вашим пользователям не нужно нажимать на точные местоположения, чтобы зарегистрировать событие клика. Это может стать более удобным для пользователя при работе с более мелкими или более сложными фигурами.

var starA = new Konva.Star({  x: 125,  y: 125,  innerRadius: 25,  outerRadius: 75,  rotation: 90,  fill: "blue",  stroke: "black",  name: "Blue Star",  hitFunc: function(context) {    context.beginPath();    context.arc(0, 0, this.getOuterRadius(), 0, Math.PI * 2, true);    context.closePath();    context.fillStrokeShape(this);  }});var ringA = new Konva.Ring({  x: 475,  y: 275,  innerRadius: 25,  outerRadius: 75,  fill: "brown",  stroke: "black",  name: "Brown Ring",  hitFunc: function(context) {    context.beginPath();    context.arc(0, 0, this.getOuterRadius(), 0, Math.PI * 2, true);    context.closePath();    context.fillStrokeShape(this);  }});

Заключительные мысли

В этом уроке мы рассмотрели различные mobile и desktop события, которые вы можете привязать к любой форме в Konva. Вы можете прикрепить эти события по одному или несколько сразу. Konva также позволяет запускать собственные пользовательские события программно с помощью метода fire(). В последнем разделе учебника показано, как определить собственные области попадания, чтобы обнаружить попадания в область, которая может быть больше или меньше исходной формы.

Объединяя знания этого учебника с другими в серии, вы должны теперь иметь возможность рисовать различные формы на холсте и разрешать своим пользователям взаимодействовать с ними.
