Производительные анимации с использованием KUTE.js: часть-3, анимация SVG...

Code JavaScript
Tutorial

В предыдущем уроке серии показано, как анимировать различные свойства CSS любого элемента с помощью KUTE.js. Однако основной движок не позволяет вам анимировать свойства, характерные для элементов SVG. Точно так же вы не можете анимировать SVG-морфинг разных фигур или рисовать различные элементы SVG с помощью линий. Для достижения любой из этих задач вам нужно будет использовать плагин SVG KUTE.js SVG.

Прежде чем мы начнем, имейте в виду, что вам нужно будет включить как основной движок KUTE.js, так и плагин SVG, чтобы примеры в этом учебнике работали.

Морфинг SVG фигур

Морфинг одной фигуры SVG в другую - очень распространенная функция, с которой вы столкнетесь. Плагин KUTE.js SVG дает нам все, что нам нужно, чтобы легко создавать свои собственные морфинговые анимации.

Существует три способа преобразования SVG-фигур с использованием этой библиотеки:

  1. Вы можете использовать метод fromTo(), чтобы указать как начальный, так и конечный путь SVG для вашего элемента.
  2. Вы также можете использовать метод to() и не указывать начальный путь. В этом случае начальное значение для морфинга будет определяться на основе значения атрибута d выбранного элемента, который вы хотите преобразовать.
  3. Еще один вариант, который у вас есть, - передать конечный путь в виде строки непосредственно в анимацию. Таким образом, вы можете избежать наличия двух разных путей в вашем SVG.
KUTE.fromTo("#shape-a", {path: "#shape-a" }, { path: "#shape-b" });KUTE.to("#shape-a", { path: "#shape-b" });KUTE.fromTo("#shape-a", {path: "#shape-a" }, { path: "The path of #shape-b as a valid string." });KUTE.to("#shape-a", { path: "The path of #shape-b as a valid string." });

Во время инициализации библиотека отображает некоторые точки на основе путей, которые мы предоставили. Эти точки затем сохраняются в двух разных массивах. Наконец, эти массивы используются для интерполяции. Существует несколько параметров, которые вы можете настроить для управления морфинговым поведением для разных путей.

  • morphPrecision: как вы могли догадаться, этот параметр позволяет указать точность морфинга. Оно указывается в виде числа, и меньшее значение означает более высокую точность. Имейте в виду, что более высокая точность приведет к большей точности, но это также будет вредно для производительности. Этот параметр не применяется, когда вы имеете дело с полигональными фигурами или путями, где атрибут d состоит только из h, l и v. В таких случаях вместо выборки новых образцов используются исходные полигональные пути.
  • reverseFirstPath: вы можете установить значение этой опции в true, чтобы изменить траекторию рисования для вашей первой фигуры. Его значение по умолчанию - false.
  • reverseSecondPath: вы можете установить значение этой опции в true, чтобы изменить траекторию рисования для вашей второй фигуры. Его значение по умолчанию также false.
  • morphIndex: Иногда точки на пути, должны покрывать большое расстояние во время морфинга. Вы можете контролировать это поведение, используя параметр morphIndex. Когда указано, этот параметр позволяет вращать конечный путь таким образом, чтобы все точки перемещались на минимальное расстояние.

Давайте используем то, что мы узнали , чтобы превратить значок батареи в значок закладки. Следует отметить, что я использовал нижний регистр l для указания пути в относительных терминах. Это требуемая разметка:

<path id="battery-a"       d="M50,10 l150,0 l0,25 l20,0 l0,50 l-20,0 l0,25 l-150,0 l0,-100z"/><path id="bookmark-a"      d="M70,10 l0,125 l40,-40 l40,40 l0,-125 l0,0 l0,0 l0,0 l0,0z"/>

Следующий JavaScript создает объект tween и запускает анимацию при нажатии кнопки:

var morphA = KUTE.to(    "#battery-a",     { path: "#bookmark-a" },    { duration: 5000 });startButton.addEventListener(  "click",  function() {    morphA.start();  },  false);

Вот демонстрация, которая показывает приведенный выше код в действии. Я также добавил дополнительный элемент, в котором анимация морфинга устанавливает reverseFirstPath в true. Это поможет вам понять общее влияние различных параметров конфигурации на морфинг. Продолжительность анимации установлена на 5 секунд, чтобы вы могли внимательно следить за анимацией и выявлять различия.

В предыдущем примере основной путь не имел никаких подпутей. Это сделало морфинг очень простым. Однако это может быть не всегда так.

Давайте добавим дополнительный подпуть к нашей закладке, а также значок аккумулятора. Если вы теперь измените иконки, вы увидите, что только первый подпункт анимируется. Второй подпуть просто исчезает в начале анимации и появляется в конце. Единственный способ анимировать все подпути в таких случаях-это изменить подпути на отдельные пути. Вот пример:

<!-- Before --><path id="battery-a"      d="M50,10 l150,0 l0,25 l20,0 l0,50 l-20,0 l0,25 l-150,0 l0,-100z             M70,30 l60,65 l-10,-65 l60,65z"/><path id="bookmark-a"      d="M70,10 l0,125 l40,-40 l40,40 l0,-125 l0,0 l0,0 l0,0 l0,0z         M80,80 l30,-45 l30,45 l0,0z"/> <!-- After --><path id="battery-b1"      d="M250,10 l150,0 l0,25 l20,0 l0,50 l-20,0 l0,25 l-150,0 l0,-100z"/><path id="battery-b2"       d="M270,30 l60,65 l-10,-65 l60,65z"/><path id="bookmark-b1"      d="M270,10 l0,125 l40,-40 l40,40 l0,-125 l0,0 l0,0 l0,0 l0,0z"/><path id="bookmark-b2"      d="M280,80 l30,-45 l30,45 l0,0z"/>

Анимация линий обводки SVG

Другой популярный эффект анимации, связанный с SVG, начинается из ничего, а затем рисуется предопределенная форма с использованием линий обводки SVG. Это можно использовать для анимации рисунка логотипов или других объектов. В этом разделе вы узнаете, как использовать KUTE.js для создания анимации значка велосипеда Font Awesome.

Существует три способа анимации линий SVG в KUTE.js. Вы можете анимировать от 0% до 100%, установив значения fromTo как 0% 0% и 0% 100%. Вы также можете нарисовать часть формы SVG, установив значения примерно на 0% 5% и 95% 100%. Наконец, вы можете установить конечное значение 0% 0%, чтобы создать эффект стирания вместо эффекта рисования.

Вот код JavaScript, который я использовал для анимации нашего велосипеда:

var wholeAnimation = KUTE.fromTo(  "#icon",  { draw: "0% 0%" },  { draw: "0% 100%" },  { duration: 10000});var partialAnimation = KUTE.fromTo(  "#icon",  { draw: "0% 5%" },  { draw: "95% 100%" },  { duration: 10000});var eraseAnimation = KUTE.fromTo(  "#icon",  { draw: "0% 100%" },  { draw: "0% 0%" },  { duration: 5000});

Как вы можете видеть в приведенном ниже примере, вам не нужно беспокоиться о нескольких подпутях внутри пути. KUTE.js анимирует все эти подпути индивидуально без каких-либо проблем. Длительность анимации используется для определения времени анимации самого длинного пути. Продолжительность хода для остальных подпутей затем определяется на основе их длины.

Анимация SVG Transforms

Мы уже узнали, как анимировать значения преобразования CSS во втором учебнике серии. Плагин KUTE.js SVG также позволяет вам использовать атрибут svgTransform, чтобы вращать, пермещать, масштабировать или искажать различные элементы SVG на веб-странице.

Атрибут rotate принимает одно значение, определяющее угол поворота. По умолчанию вращение происходит вокруг центральной точки элемента, но вы можете указать новый центр вращения, используя атрибут transformOrigin.

Атрибут translate принимает значения в формате translate: [x, y] или translate: x. При наличии одного значения значение y считается равным нулю.

При искажении элементов вам придется использовать skewX и skewY. В SVG нет поддержки skew[x, y]. Аналогично, атрибут scale также принимает только одно значение. Такое же значение используется для масштабирования элементов в направлениях x и y.

Вот фрагмент кода, который применяет все эти преобразования к прямоугольнику и кругу.

var rotation = KUTE.allTo(  "rect, circle",  { svgTransform: { rotate: 360 } },  { repeat: 1, yoyo: true });var scaling = KUTE.allTo(  "rect, circle",  { svgTransform: { scale: 1.5 } },  { repeat: 1, yoyo: true });var translation = KUTE.allTo(  "rect, circle",  { svgTransform: { translate: [100, -50] } },  { repeat: 1, yoyo: true });var skewing = KUTE.allTo(  "rect, circle",  { svgTransform: { skewX: 25 } },  { repeat: 1, yoyo: true });

Я установил параметр yoyo в true, чтобы после воспроизведения анимации в обратном направлении свойства преобразования были установлены на их начальное значение. Таким образом, мы можем повторять анимацию снова и снова, нажимая на кнопки.

Если вы нажмете кнопку Rotate в демо, вы заметите, что это не влияет на круг. Чтобы наблюдать вращение круга, вам нужно будет применить к нему косое преобразование, чтобы изменить его форму, а затем сразу же повернуть.
