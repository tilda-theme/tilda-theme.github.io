Быстрая и простая документация с использованием-Markdown...

Web Design Workflow
Tutorial

Итак, вы создали отличную тему, шаблон или веб-приложение. Теперь для утомительной части документация. Написание контента не обязательно будет сложной задачей, скорее всего, создание файлов справки займет много драгоценного времени. Markdown, легкий и *действительно* простой синтаксис форматирования который может решить все это за вас.

О Markdown

Markdown, созданный Джоном Грубером, представляет собой простой синтаксис форматирования текста, который значительно упрощает создание основных элементов HTML.

Вместо использования HTML-тегов (которые, собственно говоря, занимают много времени для написания), работа Markdown состоит в том, чтобы уйти с пути вашего мыслительного процесса, позволяя вам сосредоточиться на контенте. Поскольку синтаксис настолько прост, его легко писать и читать с Markdown. Позже в этом уроке мы рассмотрим шаги по созданию документации темы с использованием Markdown, поэтому вы увидите, насколько она легкая и быстрая!

Преобразование Markdown

Когда вы освоите написание Markdown, вам понадобится какое-то приложение для анализа, чтобы преобразовать Markdown в HTML-разметку.

Оригинальный конвертер Markdown был создан в Perl, но с тех пор появилось много приложений (на разных платформах). Давайте рассмотрим некоторые варианты, как онлайн, так и в вашей собственной системе.

Dingus (онлайн)

Dingus - это веб-приложение, созданное теми же людьми, которые создали вам Markdown. Просто скопируйте и вставьте свой код Markdown из текстового редактора (или введите его в текстовую область), и одним нажатием кнопки появится ваш HTML-код (а также предварительный просмотр). Ничего особенного, но это простой способ конвертировать Markdown в HTML.

MarkdownPad (Windows)

MarkdownPad - это большое приложение для Windows, которое позволяет вам легко создавать файлы Markdown (и это бесплатно!) Она включает в себя удивительные возможности как предпросмотр HTML, легкое форматирование с горячими клавишами, настройки CSS, HTML экспорта.

Mou (OS X)

Mou - отличное бесплатное приложение для Mac, которое позволяет писать Markdown простым и красивым способом. Он имеет отличные функции, такие как пользовательские стили, сочетания клавиш, предварительный просмотр HTML, экспорт HTML (с дополнительным стилем CSS), экспорт PDF, поддержка диктовки и многое другое. В этом уроке мы будем использовать Mou для создания документации нашей темы.

MarkdownNote (OS X & iOS)

MarkdownNote - еще одно отличное приложение для написания Markdown, и приложение для iOS идеально подходит, если вы хотите написать Markdown на ходу. Как и в других приложениях, которые мы рассмотрели, в нем есть несколько замечательных функций, в том числе предварительный просмотр HTML, сочетания клавиш и синхронизация Dropbox.

Создание темы документации

Итак, мы рассмотрели, что такое Markdown, и рассмотрели некоторые инструменты, которые можно использовать для написания Markdown. Теперь давайте создадим некоторую документацию темы для несуществующей темы, охватывающую то, что вы обычно должны включать в документацию.

Синтаксис Markdown

На следующих шагах мы рассмотрим Markdown, поскольку он соответствует нашим потребностям. Для более подробного ознакомления с синтаксисом Markdown ознакомьтесь с разделом Markdown: входы и выходы в Nettuts+.

Что должен включать файл документации?

Поскольку вы уже знаете входы и выходы своей темы/шаблона/веб-приложения, изучение основ может показаться немного утомительным. Цель файла документации - служить руководством для других пользователей и покупателей. Часто требуется установка и некоторая настройка, о которой пользователям необходимо знать - и ваша работа - обучать их. Это то, что мы могли бы включить:

  • Информация
    • Автор
    • Дата создания
    • Версия
    • Контактная информация.
  • Файловая структура
    • HTML
    • CSS
    • JavaScript
  • Индивидуальная файловая структура
    • Структура HTML
    • Структура CSS
  • Пользовательские стили
  • Настройки файлов
    • Настройки CSS
    • Настройки Javascript
  • Совместимость браузера
  • История изменений

Помните, что документация должна быть достаточно простой для того, чтобы любой, кто обладает базовыми знаниями, мог понять, как изменять важные файлы. Например, вам не обязательно показывать, как изменить определенные значения CSS, но вы должны показать, как получить доступ к этим файлам.

Шаг 1: Настройка файла Markdown

Теперь пришло время веселья! Открой редактор Markdown (я буду использовать Mou для Mac). Создайте заголовок для вашего файла документации:

#Theme Documentation

Элементы заголовков можно записать, просто добавив от одного до шести "#" перед содержимым. В приведенном выше примере мы использовали один знак для создания элемента h1 с текстом "Theme Documentation". Если вы хотите создать элемент h3, вы должны написать ###Heading 3.

Теперь давайте создадим горизонтальное правило, чтобы отделить заголовок от другого содержимого. Опять же, в Markdown это очень просто:

***

Шаг 2: Информация о теме

Важным раздел, который нужно добавить в свою документацию, является информационный раздел.

*Theme Name:* Theme*Author:* Suhail Dawood*Created:* 08/08/2012*Version:* 1.0.1***Thank you for your purchase! If you have any questions about this theme, feel free to e-mail me at **email@tutsplus.com** or tweet me **@tutsplus*****

Давайте посмотрим на HTML-эквивалент кода Markdown:

<em>Theme Name: </em>Theme<br /><em>Author: </em>Suhail Dawood<br /><em>Created: </em>08/08/2012<br /><em>Version: </em>1.0.1<br /><hr />Thank you for your purchase! If you have any questions about this theme, feel free to e-mail me at <strong>email@tutsplus.com</strong> or tweet me <strong>@tutsplus</strong><hr />

Просто взглянув на два разных источника, вы сразу увидите, что Markdown гораздо более понятен для понимания и изменения. Вместо того, чтобы постоянно добавлять < и >, Markdown позволяет вам сосредоточиться на контенте. Чтобы подчеркнуть, добавьте звездочку до и после текста (например, *Подчеркнутый текст*). Чтобы выделить, добавьте две звездочки до и после текста (например, **Облегченный текст**). Чтобы добавить разрыв строки, просто добавьте два пробела в точку, в которую вы хотите вставить разрыв строки.

Шаг 3: Добавление оглавления

Теперь давайте добавим список содержимого в наш файл Markdown:

##Table of Contents1. HTML Structure2. CSS Files3. Javascript Files4. PSD Files5. Theme Styles6. Tweaking Javascript7. Tweaking CSS8. Browser Compatibility***

Если бы мы создали это в HTML, вот как это будет выглядеть:

<h2>Table of Contents</h2><ol>   <li>HTML Structure</li>   <li>CSS Files</li>   <li>Javascript Files</li>   <li>PSD Files</li>   <li>Theme Styles</li>   <li>Tweaking Javascript</li>   <li>Tweaking CSS</li>   <li>Browser Compatibility</li></ol><hr />

Вместо того, чтобы создавать упорядоченный список и отдельные элементы списка, просто напишите 1. First Element и продолжайте добавлять следующие элементы.

Шаг 4: структура HTML

Важно, чтобы ваши пользователи размещали файлы сайта. Поскольку мы создаем документацию для темы, мы должны указать, как структурирован HTML-код темы. Мы можем об этом сообщить с помощью следующего кода:

##1. HTML StructureThe HTML Structure for each page is as follows:* Meta* Link to CSS Files* Header  * Logo* Social Links* Navigation* Main Content* Content Slider* Articles* Sidebar* Search* Post Archives* Mailing List* Link to Javascript Files* Javascript***

Это просто. Для создания оглавления мы использовали упорядоченный список. В этом разделе мы использовали вложенные неупорядоченные списки. Чтобы создать неупорядоченный список в Markdown, просто добавьте звездочку и пробел перед текстом (например, *Item 1). Чтобы вложить элементы списка, просто сделайте отступ внутрь и создайте еще один элемент списка.

Шаг 5: CSS файлы

Документация по CSS очень важна в темах. Хорошая идея - сообщить в общих чертах различные CSS-файлы, включенные в тему, а также краткое описание каждого файла:

##2. CSS FilesThere are 3 CSS files in this theme:* main.css* colors.css* skeleton.css#####main.cssThis CSS file is the main stylesheet for the theme. It holds all the values for the different elements of theme and the default color scheme.#####colors.cssThis CSS file holds the styling of all the other colors schemes that are included in the theme.#####skeleton.cssThis CSS file allows the theme to be responsive, adapting to different screen sizes. ***

Обязательно используйте заголовки для разделения различных разделов файла документации. Хорошо структурированный файл документации приведет к меньшему количеству запутанных пользователей!

Шаг 6: Файлы Javascript

Если ваша тема включает в себя файлы Javascript, хорошей идеей будет, выделить их в документации:

##3. Javascript FilesThere are 2 Javascript files in this theme:* jquery.js* slider.js#####jquery.jsThis theme uses the jQuery Javascript library.#####slider.jsThe content slider in the theme runs on this Javascript file. There are some settings that can be tweaked, this will be covered in the "Tweaking Javascript" section.***

Убедитесь, что вы не только перечислите, но и опишите файлы в вашей теме. Это облегчит пользователю понимание содержимого каждого файла и решит, хотят ли они возиться с ним.

Шаг 7: PSD файлы

Несмотря на то, что в ThemeForest есть отдельный раздел для шаблонов PSD, многие авторы решили включить PSD-файлы в свои кодированные шаблоны, чтобы предоставить пользователю свободу вносить изменения в дизайн. Если в вашей теме есть PSD-файлы, было бы неплохо охарактеризовать их в общих чертах так же, как мы сделали со всеми остальными файлами:

##4. PSD FilesIncluded in this theme is 5 different PSD files:1. homepage.psd2. about.psd3. portfolio.psd4. blog.psd5. contact.psdThese PSD files will be handy if you would like to make any changes to the theme"s design. You can also create a new page layout using the existing PSD files as a base to work with.***

Лучше четко указывать свои PSD-файлы (например, full-width-page.psd), а не расплывчатые имена (например, template-003.psd). Таким образом, пользователи могут найти то, что им нужно, без необходимости открывать каждый файл.

Шаг 8: Стили тем

Скорее всего, ваша тема будет включать в себя выбор цветовых схем. Если это так, убедитесь, что вы обрисовали в общих чертах, как это делается:

##5. Theme StylesIncluded with this theme are 10 different theme styles:1. Light2. Dark3. Grey4. Green5. Red6. Orange7. Blue8. Purple9. Brown10. Dark BlueTo change these styles, go to the WordPress backend, select **Options > Styles** and select the style you would like.***

В приведенном выше примере я перечислил различные цветовые схемы, включенные в тему, и показал, как их менять.

Шаг 9: настройка Javascript

Если ваш код содержит файлы Javascript, которые имеют параметры настройки (например, скрипт-слайдер), было бы неплохо сообщить вашим пользователям, как управлять этими значениями:

##6. Tweaking JavascriptThe content slider in the theme runs off of slider.js, and there are a couple of values that can be changed to alter the look and feel of the slider.#####Changing ValuesIn slider.js, you can alter these values:<code>auto: true</code>  *Boolean: Animate automatically, true or false*  <code>speed: 1000</code>  *Integer: Speed of the transition, in milliseconds*<code>pager: true</code>  *Boolean: Show pager, true or false*  <code>nav: false</code>  *Boolean: Show navigation, true or false*  ***

Приведенный выше код описывает, как пользователь может изменять значения. Чтобы стилизовать код, поместите соответствующий текст в теги <code>. Такие приложения, как Mou, добавляют стили к этим элементам в режиме предварительного просмотра, а также можно экспортировать код, чтобы сохранить стилизацию. Мы немного поговорим об экспорте документации позже.

CSS-файлы очень часто настраиваются пользователем. Они, безусловно, посчитают полезным, если вы добавите в документ раздел, показывающий, как получить доступ к файлам CSS для редактирования.

Экспорт документации

Теперь, когда мы завершили создание файла документации, пришло время преобразовать Markdown в HTML или PDF файл. Я буду использовать Mou для экспорта своей документации, но шаги являются общими и применимы к большинству приложений.

Экспорт в HTML

Чтобы экспортировать документацию в HTML, просто выберите File > Export > HTML. Назовите файл документации и отметьте/снимите флажок "Включить CSS" в зависимости от того, хотите ли вы, чтобы к вашему HTML-файлу применялся стиль, показанный в приложении. Мы не будем создавать отдельный файл CSS, но вместо этого добавим стиль в файл HTML. Mou также создает необходимые теги, такие как html, doctypehead и т.д.

Экспорт в PDF

Если вы хотите экспортировать документацию в PDF, выберите File > Export > PDF. В случае с Mou все стили, показанные в предварительном просмотре HTML, будут включены в файл PDF.

И мы это сделали! Предельно ясная, читаемая документация для клиентов, заказчиков и коллег. Загрузите исходные файлы для этого урока и ознакомьтесь с получающийся в результате файлами .md .html и .pdf.
