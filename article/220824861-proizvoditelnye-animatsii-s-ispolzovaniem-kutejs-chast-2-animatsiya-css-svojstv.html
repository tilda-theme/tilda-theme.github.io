Производительные анимации с использованием KUTE.js: часть 2, Анимация CSS Свойств...

Code JavaScript
Tutorial

Первый учебник серии сосредоточен на предоставлении начинающему дружественного введения в KUTE.библиотеку JS. В этом учебнике мы анимировали только свойство opacity и rotateZ для наших элементов. В этом уроке вы узнаете, как оживить остальные свойства CSS с помощью KUTE.js.

Некоторые из этих свойств потребуют загрузки плагина CSS, в то время как другие могут быть анимированы с использованием самого ядра. Оба этих сценария будут обсуждаться отдельно в учебнике.

Свойства Box Model

Ядро KUTE.js может анимировать только самые распространенные свойства box model: width, height, top, и left. Вам нужно будет использовать плагин CSS для анимации почти всех других свойств box model. Вот пример, который анимирует верхнее положение, ширину и высоту наших квадратов из предыдущего учебника:

var animateTop = KUTE.allFromTo(  theBoxes,  { top: 0 },  { top: 100 },  { offset: 100 });var animateA = KUTE.fromTo(  boxA,  { height: 100 },  { height: 175 });var animateB = KUTE.fromTo(  boxB,  { width: 100 },  { width: 200 });

Возможно, вы заметили, что я использовал allFromTo() для анимации верхнего свойства всех квадратов. Однако я использовал fromTo() для анимации отдельных квадратов. Вы должны помнить, что квадраты остаются в своем последнем состоянии после завершения анимации.

С помощью плагина CSS вы сможете анимировать margin, padding, и borderWidth. После того как вы включите плагин в свой проект, остальная часть процесса будет точно такой же.

Анимация Свойств Преобразования

Вы можете анимировать почти все свойства преобразования, упомянутые в спецификации, с помощью самого ядра. В этом случае нет необходимости загружать плагин CSS.

Вы можете анимировать перемещение элементов в 2D-пространстве, используя translate. Аналогично, вы можете использовать translateX, translateY и translateZ, чтобы анимировать элементы вдоль соответствующих осей. Чтобы свойство translateZ имело какой-либо эффект, вам также нужно будет установить значение для свойства parentPerspective. Вот пример:

var animateAll = KUTE.allFromTo(  theBoxes,  { translateY: 0 },  { translateY: 100 },  { offset: 1000 });var animateA = KUTE.fromTo(  boxA,  { translateZ: 0 },  { translateZ: 50 },  { parentPerspective: 100, parentPerspectiveOrigin: "0% 0%" });var animateB = KUTE.fromTo(  boxB,  { translateX: 0 },  { translateX: -200 });startButton.addEventListener(  "click",  function() {    animateAll.start();    animateA.start();    animateB.start();  },  false);

Если вы нажмете кнопку Start Animation и внимательно наблюдаете за анимацией, вы увидите, что красный квадрат перемещается сначала на -200 в направлении X. После этого он перемещается в исходное положение и начинает перемещаться в направлении Y. Причина того, что квадрат сначала анимируется по translateX состоит в том, что мы добавили задержку для анимации translateY, используя свойство offset.

Вы также можете выполнять поворот вдоль определенной оси с помощью функций rotate, rotateX, rotateY и rotateZ. Поскольку rotateX и rotateY являются 3D-вращениями, вам придется использовать свойство perspective для анимации вращения, чтобы она работала как ожидалось. В следующем примере показано, как использование свойства perspective влияет на общую анимацию для этих двух свойств.

var animateAll = KUTE.allFromTo(  theBoxes,  { rotate: 0 },  { rotate: 360 },  { offset: 1000 });var animateA = KUTE.fromTo(  boxA,  { rotateY: 0 },  { rotateY: 180 },  { perspective: 100 });var animateB = KUTE.fromTo(  boxB,  { rotateY: 0 },  { rotateY: -180 });startButton.addEventListener(  "click",  function() {    animateAll.start();    animateA.start();    animateB.start();  },  false);

В приведенном выше примере квадрат A и квадрат B начинают свое вращение вдоль оси Y в одно и то же время, но полученная анимация очень отличается из-за свойства perspective. Возможно, вы заметили, что оранжевый квадрат не выполняет поворот вокруг его центра, который был применен к нему, используя animateAll. Это связано с тем, что по умолчанию все анимации имеют продолжительность 500 миллисекунд, и мы одновременно применяем оба варианта animateAll и animateA на оранжевом поле.

Поскольку animateA применяется после animateAll, его анимация имеет приоритет над animateAll. Вы увидите, что общее вращение с использованием animateAll по-прежнему применяется на оранжевом поле после увеличения продолжительности анимации. Другими словами, вы не можете анимировать различные свойства преобразования, используя одновременно несколько объектов tween. Все свойства преобразования, которые вы хотите анимировать, должны быть указаны внутри одного объекта tween. В следующем примере должно быть ясно:

// This will not work as expectedvar translateAll = KUTE.allFromTo(  theBoxes,  { translateY: 0 },  { translateY: 100 },  { offset: 1000 });var rotateAll = KUTE.allFromTo(  theBoxes,  { rotate: 0 },  { rotate: 360 },  { offset: 1000 });startButton.addEventListener(  "click",  function() {    translateAll.start();    rotateAll.start();  },  false);// This will work as expectedvar rtAll = KUTE.allFromTo(  theBoxes,  { translateY: 0, rotate: 0 },  { translateY: 100, rotate: 360 },  { offset: 1000 });startButton.addEventListener(  "click",  function() {    rtAll.start();  },  false);

Использование плагина CSS

Как я упоминал ранее, не все свойства CSS могут быть анимированы с использованием только основного ядра KUTE.js. Вам нужно использовать дополнительный плагин CSS для анимации свойств, таких как padding, margin, background position изображений и другие свойства связанные с border. Поэтому перед тем, как вы попытаетесь использовать какой-либо из примеров в этом разделе, вы должны включить плагин в свой проект.

<script src="https://cdn.jsdelivr.net/kute.js/1.6.2/kute-css.min.js"></script>

После того, как вы включили плагин, вы сможете анимировать свойство border-radius с помощью borderRadius. Вы также можете анимировать все значения border-radius с помощью borderTopLeftRadius, borderTopRightRadius, borderBottomLeftRadius и borderBottomRightRadius. Вот пример, показывающий анимацию в действии.

Если вы нажмете кнопку Start Animation, вы заметите, что верхний левый border-radius для красного и желтого квадратов анимируется после задержки. Это происходит из-за свойства offset. Остальные радиусы анимируются, как только мы нажимаем на кнопку. Приведенный выше пример был создан с помощью следующего кода:

var animateAll = KUTE.allFromTo(  theBoxes,  { borderTopLeftRadius:"0%" },  { borderTopLeftRadius:"100%" },  { offset: 1000 });var animateA = KUTE.fromTo(  boxA,  { borderTopRightRadius:"0%" },  { borderTopRightRadius:"100%" });var animateB = KUTE.fromTo(  boxB,  { borderBottomLeftRadius:"0%" },  { borderBottomLeftRadius:"100%" });var animateC = KUTE.fromTo(  boxC,  { borderBottomRightRadius:"0%" },  { borderBottomRightRadius:"100%" });startButton.addEventListener(  "click",  function() {    animateAll.start();    animateA.start();    animateB.start();    animateC.start();  },  false);

Мы не связали объекты анимации вместе, поэтому все анимации начинаются сразу на этот раз. Вы также можете анимировать цвет разных границ аналогичным образом, используя borderColor, borderTopColor, borderLeftColor, borderBottomColor и borderRightColor.

Заключительные мысли

В этом уроке мы узнали о различных свойствах CSS, которые можно анимировать с использованием и без использования плагина KUTE.js CSS. Если у вас есть какие-либо вопросы, пожалуйста, напишите об этом мне в комментариях.
