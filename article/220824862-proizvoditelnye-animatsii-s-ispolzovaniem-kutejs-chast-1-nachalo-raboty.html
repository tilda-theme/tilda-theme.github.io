Производительные анимации с использованием KUTE.js: часть 1, Начало работы...

Code JavaScript
Tutorial

KUTE.js - это механизм анимации на основе JavaScript, который фокусируется на производительности и эффективности памяти при анимации различных элементов на веб-странице. Я уже написал серию об использовании Anime.js для создания анимации на основе JavaScript. На этот раз мы узнаем о KUTE.js и о том, как мы можем использовать его для анимации свойств CSS, SVG и текстовых элементов, среди прочего.

Установка

Прежде чем мы перейдем к некоторым примерам, давайте сначала установим библиотеку. У KUTE.js есть основной движок, а также есть плагины для анимации значений различных свойств CSS, атрибутов SVG или текста. Вы можете напрямую ссылаться на библиотеку из популярных CDN, таких как cdnjs и jsDelivr.

<script src="https://cdnjs.cloudflare.com/ajax/libs/kute.js/1.6.2/kute.min.js"></script><script src="https://cdn.jsdelivr.net/kute.js/1.6.2/kute.min.js"></script>

Вы также можете установить KUTE.js с помощью NPM или Bower с помощью следующих команд:

npm install --save kute.jsbower install --save kute.js

После включения библиотеки в проекты можно приступать к созданию собственных анимационных последовательностей.

Объект Tween

При создании анимации с помощью KUTE.js вам нужно определить объекты tween. Эти объекты tween предоставляют всю информацию, связанную с анимацией, для данного элемента или элементов. Сюда входят сам элемент, свойства, которые требуется анимировать, длительность анимации и другие атрибуты, такие как число повторов, задержка или смещение.

Вы можете использовать метод .to() или метод .fromTo() для анимации набора свойств CSS от одного значения к другому. Метод .to() анимирует свойства от их значения по умолчанию или их вычисленного / текущего значения до конечного заданного значения. В случае метода .fromTo() , вы должны предоставить как начальные и конечные значения анимации.

Метод .to() полезен, когда вы не знаете текущее значение или значение по умолчанию для свойства, которое вы хотите анимировать. Одним из основных недостатков этого метода является то, что библиотека должна сама вычислять текущее значение всех свойств. Это приводит к задержке в несколько миллисекунд после вызова .start() для запуска анимации.

Метод .fromTo() позволяет указать начальные и конечные значения анимации самостоятельно. Это может незначительно улучшить производительность анимации. Теперь вы можете самостоятельно задать единицы измерения для начального и конечного значений и избежать неожиданностей во время анимации. Один из недостатков использования .fromTo() заключается в том, что вы не сможете укладывать несколько свойств преобразования в цепочку. В таких случаях вам придется использовать метод .to().

Помните, что оба .fromTo() и .to() предназначены для использования при анимации отдельных элементов. Если вы хотите анимировать несколько элементов одновременно, вам придется использовать либо .allTo(), либо .allFromTo(). Эти методы работают так же, как и их одноэлементные аналоги и наследуют все их атрибуты. Они также получают дополнительный атрибут offset, определяющий задержку между запуском анимации для различных элементов. Это смещение определяется в миллисекундах.

Вот пример, который анимирует непрозрачность трех разных квадратов в последовательности.

Следующий код JavaScript используется для создания последовательности анимации :

var theBoxes = document.querySelectorAll(".box");var startButton = document.querySelector(".start");var animateOpacity = KUTE.allFromTo(  theBoxes,  { opacity: 1 },  { opacity: 0.1 },  { offset: 700 });startButton.addEventListener(  "click",  function() {    animateOpacity.start();  },  false);

Все вышеприведенные поля имеют класс box, который использовался для их выбора с помощью метода querySelectorAll(). Метод allFromTo() в KUTE.js используется для анимации непрозрачности этих полей от 1 до 0,1 со смещением 700 миллисекунд. Как вы можете видеть, объект tween не запускает анимацию сам по себе. Вы должны вызвать метод start(), чтобы запустить анимацию.

Управление воспроизведением анимации

В предыдущем разделе мы использовали метод start(), чтобы начать анимацию. Библиотека KUTE.js также предоставляет несколько других методов, которые можно использовать для управления воспроизведением анимации.

Например, вы можете остановить любую анимацию, которая в настоящее время выполняется с помощью метода stop(). Имейте в виду, что вы можете использовать этот метод, чтобы остановить анимацию только тех объектов tween, которые были сохранены в переменной. Анимация для любого объекта твина, созданного «на лету», не может быть остановлена этим методом.

У вас также есть возможность просто приостановить анимацию с помощью метода pause(). Это полезно, если вы хотите снова возобновить анимацию позднее. Вы можете использовать resume() или play(), чтобы возобновить любую анимацию, которая была приостановлена.

Следующий пример - это обновленная версия предыдущей демонстрации со всеми четырьмя добавленными к ней методами.

Вот код JavaScript, необходимый для добавления функций запуска, остановки, воспроизведения и паузы.

var theBoxes = document.querySelectorAll(".box");var startButton = document.querySelector(".start");var stopButton = document.querySelector(".stop");var pauseButton = document.querySelector(".pause");var resumeButton = document.querySelector(".resume");var animateOpacity = KUTE.allFromTo(  theBoxes,  { opacity: 1 },  { opacity: 0.1 },  { offset: 700, duration: 2000 });startButton.addEventListener(  "click",  function() {    animateOpacity.start();  },  false);stopButton.addEventListener(  "click",  function() {    animateOpacity.stop();  },  false);pauseButton.addEventListener(  "click",  function() {    animateOpacity.pause();  },  false);resumeButton.addEventListener(  "click",  function() {    animateOpacity.resume();  },  false);

Я изменил продолжительность анимации до 2000 миллисекунд. Это дает нам достаточно времени, чтобы нажимать разные кнопки и видеть, как они влияют на воспроизведение анимации.

Сцепление Tweens вместе

Вы можете использовать метод chain() для связывания различных tweens вместе. После того, как различные tweens были соединены, они вызывают метод start() на других tweens после того, как их собственная анимация закончилась.

Таким образом, вы можете воспроизводить различные анимации в разной последовательности. Вы можете сцеплять различные tweens друг с другом для того, чтобы проигрывать их в цикле. В следующем примере должно быть ясно:

var animateOpacity = KUTE.allFromTo(  theBoxes,  { opacity: 1 },  { opacity: 0.1 },  { offset: 100, duration: 800 });var animateRotation = KUTE.allFromTo(  theBoxes,  { rotate: 0 },  { rotate: 360 },  { offset: 250, duration: 800 });opacityButton.addEventListener(  "click",  function() {    animateOpacity.start();  },  false);rotateButton.addEventListener(  "click",  function() {    animateRotation.start();  },  false);chainButton.addEventListener(  "click",  function() {    animateOpacity.chain(animateRotation);    animateOpacity.start();  },  false);loopButton.addEventListener(  "click",  function() {    animateOpacity.chain(animateRotation);    animateRotation.chain(animateOpacity);    animateOpacity.start();  },  false);

У нас уже был один аниматор для анимации непрозрачности. Теперь мы добавили еще один, который анимирует вращение наших квадратов. Первые две кнопки анимируют непрозрачность и вращение по одному. Третья кнопка запускает цепочку animateOpacity с animateRotation.

Сама цепочка не запускает анимацию, поэтому мы также используем метод start() для запуска анимации непрозрачности. Последняя кнопка используется для соединения двух tweens друг с другом. На этот раз анимация продолжает играть бесконечно, как только они будут запущены. Вот демо-версия CodePen, которая показывает весь приведенный выше код в действии:

Чтобы полностью понять, как работает цепочка, вам придется нажимать кнопки в определенной последовательности. Сначала нажмите кнопку Animate Opacity, и вы увидите, что анимация непрозрачности воспроизводится только один раз, а затем ничего не происходит. Теперь нажмите кнопку Animate Rotation, и вы увидите, что квадраты вращаются один раз, а затем ничего не происходит.

После этого нажмите кнопку Chain Animations, и вы увидите, что сначала отображается анимация непрозрачности, и после завершения ее итерации анимация вращения начинает играть сама по себе. Это произошло потому, что анимация вращения теперь привязана к анимации непрозрачности.

Теперь снова нажмите кнопку Animate Opacity, и вы увидите, что и непрозрачность, и вращение анимированы в последовательности. Это происходит потому, что они уже были связаны после того, как мы нажали на Chain Animations.

В этот момент нажатие кнопки Animate Rotation будет только анимировать поворот. Причиной такого поведения является то, что мы привязали анимацию вращения к анимации непрозрачности. Это означает, что поля будут вращаться каждый раз, когда непрозрачность будет анимирована, но анимация вращения не означает, что непрозрачность также будет анимирована.

Наконец, вы можете нажать кнопку Play in a Loop. Это свяжет обе анимации друг с другом, и как только это произойдет, анимации будут продолжать играть в бесконечном цикле. Это происходит потому, что конец одной анимации запускает начало другой анимации.

Заключительные мысли

В этом вводном учебнике по KUTE.js вы узнали об основах библиотеки. Мы начали с установки, а затем перешли к различным методам, которые можно использовать для создания объектов tween.

Вы также узнали, как управлять воспроизведением анимации и как объединить разные tweens вместе. Когда вы полностью поймете объединение в цепочку, вы сможете создавать интересные анимации, используя эту библиотеку.
