Шаблоны Проектирования: Декоратор...

Code Design Patterns
Tutorial

Ранее в этой серии, мы изучили шаблоны фасад и адаптер. С помощью фасада, мы можем упрощать большие системы, а путем реализации адаптера мы можем безопасно работать с внешним API и классами. Теперь мы собираемся рассмотреть шаблон проектирования декоратор, который также подпадает под категорию структурных шаблонов.

Мы можем использовать шаблон декоратор, когда мы хотим добавить некоторую дополнительную ответственность нашему базовому классу. Этот шаблон проектирования является прекрасной альтернативой созданию подклассов, которые расширяют функционал других классов.

Проблема

Если вас смущает мысль о том, что мы можем достичь такого же результата используя наследование, позвольте мне показать несколько примеров, которые прогонят ваше смущение и заставят полюбить шаблон декоратор.

Я собираюсь взять в качестве примера класс, который отвечает за генерирование содержимого письма электронной почты. В следующем блоке кода вы можете увидеть, этот класс работает очень хорошо для генерации содержимого электронной почты пока не нужно вносить какие-либо изменения.

class eMailBody {    private $header = "This is email header";  private $footer = "This is email Footer";public $body = "";public function loadBody() {$this->body .= "This is Main Email body.<br />";}}

Мы знаем, что приближается Рождество, и допустим, я хочу поприветствовать моего читателя сообщением из моей следующей рассылки. Так что я должен добавить сообщение с изображением в тело письма электронной почты.

Для этого я могу просто редактировать свой класс электронной почты, чего в действительности я конечно же не хочу делать. Таким образом можно реализовать наследование для достижения того же эффекта. Я наследую отдельный дочерний класс от главного класса письма электронной почты:

class christmasEmail extends eMailBody {    public function loadBody() {parent::loadBody();$this->body .= "Added Content for Xmas<br />";}}$christmasEmail = new christmasEmail();$christmasEmail->loadBody();echo $christmasEmail->body;

Итак мой код готов, и через несколько дней я хочу отправить электронную почту с Новогодними поздравлениями. Мы можем использовать ту же технику, как мы делали на Рождество.

class newYearEmail extends eMailBody {    public function loadBody() {parent::loadBody();$this->body .= "Added Content for New Year<br />";}}$newYearEmail = new newYearEmail();$newYearEmail->loadBody();echo $newYearEmail->body;

Все идет гладко, без каких-либо проблем. Теперь давайте представим, что я забыл поприветствовать моих посетителей в обоих случаев (Рождество и Новый Год), и я хочу отправить оба приветствия в одном сообщении электронной почты, без изменения кода в базовом классе.

На ум сразу приходит следующий вопрос: а помогут ли в этом случае под-классы и наследование? Можно пойти этим путем, но нам нужно будет использовать дополнительный/ненужный код для достижения этой цели. Можно воспользоваться трейтами, которые позволяют реализовать что-то похожее на множественное наследование.

Решение

Проблема, которую мы обсуждали в предыдущем разделе может быть решена путем реализации шаблона декоратор.

Согласно Википедии:

Шаблон декоратор (также известный как обертка) — это шаблон проектирования, который позволяет добавлять поведения для отдельного объекта, либо статически, либо динамически, не затрагивая поведение других объектов того же класса.

В предыдущем разделе мы видели, что мы можем расширить функции/поведения, используя один под-класс, но когда дело доходит до добавления нескольких функций/поведения, это становится довольно сложным. И вот, где мы должны использовать шаблон декоратор.

Интерфейс

interface eMailBody {    public function loadBody();}

Это простой интерфейс, который нужен для того чтобы убедиться, что класс реализует необходимые методы.

Основной Класс Email

class eMail implements eMailBody {    public function loadBody() {echo "This is Main Email body.<br />";} }

Это основной класс, который создает тело письма электронной почты по умолчанию, который я обычно использую для отправки писем. Однако мне нужно реализовать возможность менять содержимое, на основании некоторых событий, но при этом не изменяя код основного класса.

Основной Декоратор

abstract class emailBodyDecorator implements eMailBody {    protected $emailBody;public function __construct(eMailBody $emailBody) {$this->emailBody = $emailBody;}abstract public function loadBody();}

Это наш основной класс декоратора, который содержит ссылку на класс электронной почты и изменяет свое поведение, при необходимости. Здесь мы определили один абстрактный метод, loadBody, который другие декораторы должны реализовать, чтобы изменить поведение исходного класса.

Декораторы

class christmasEmailBody extends emailBodyDecorator {    public function loadBody() {echo "This is Extra Content for Christmas<br />";$this->emailBody->loadBody();}}class newYearEmailBody extends emailBodyDecorator {public function loadBody() {echo "This is Extra Content for New Year.<br />";$this->emailBody->loadBody();}}

Здесь мы создали два подкласса, которые фактически реализуют изменение поведения для нашего класса электронной почты.

Соединяем Все Вместе

Мы создали все необходимые элементы. Все, что нам нужно сделать, это использовать наш код и насладиться результатом.

interface eMailBody {    public function loadBody();}class eMail implements eMailBody {public function loadBody() {echo "This is Main Email body.<br />";} }abstract class emailBodyDecorator implements eMailBody {protected $emailBody;public function __construct(eMailBody $emailBody) {$this->emailBody = $emailBody;}abstract public function loadBody();} class christmasEmailBody extends emailBodyDecorator {public function loadBody() {echo "This is Extra Content for Christmas<br />";$this->emailBody->loadBody();}}class newYearEmailBody extends emailBodyDecorator {public function loadBody() {echo "This is Extra Content for New Year.<br />";$this->emailBody->loadBody();}}

Теперь мы можем использовать этот класс декоратора различными способами при необходимости:

/* *  Normal Email */$email = new eMail();$email->loadBody();// OutputThis is Main Email body./* *  Email with Xmas Greetings */$email = new eMail();$email = new christmasEmailBody($email);$email->loadBody();// OutputThis is Extra Content for ChristmasThis is Main Email body./* *  Email with New Year Greetings */$email = new eMail();$email = new newYearEmailBody($email);$email->loadBody();// OutputThis is Extra Content for New Year.This is Main Email body./* *  Email with Xmas and New Year Greetings */$email = new eMail();$email = new christmasEmailBody($email);$email = new newYearEmailBody($email);$email->loadBody();// OutputThis is Extra Content for New Year.This is Extra Content for ChristmasThis is Main Email body.

Теперь мы изменили тело электронной почты без изменения основного класса.

Заключение

Каждое приложение в определенное время нуждается в изменениях или улучшениях. Поэтому в таком случае мы можем реализовать шаблон проектирования декоратор и он в конечном итоге поможет улучшить качество кода, сделав его более расширяемым.

Это была моя попытка дать пояснения о шаблоне проектирования декоратор, но если у вас есть дополнительные комментарии или вопросы, пожалуйста не стесняйтесь добавлять их ниже.
