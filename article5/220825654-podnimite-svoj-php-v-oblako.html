Поднимите свой PHP в облако ...

Code PHP
Tutorial

Эта статья проведет вас через облачный хостинг для вашего PHP-приложения с Pagoda Box. В ней подробно рассказывается, как обрабатывать зависимости PHP, получать и запускать базы данных MySQL (и отлаживать локально), а также легко настраивать различные аспекты PHP.

Введение

Как разработчик, который играл с PHP с 11 лет, трудно вспомнить многие вещи, которые полностью изменили мой подход к разработке. В 2007 году я открыл для себя CodeIgniter, что заставило меня переосмыслить, как я структурирую свой код и использовать ООП. В 2010 году я начал работать с Git и Git Flow для совместной работы и ветвления, но в последнее время самое важное, что нужно изменить в процессе разработки, - это одна из «Платформы как службы», называемая Pagoda Box.

Вы можете быть разработчиком Rockstar, который знает, как настроить стек LAMP, но сколько вашего потенциального времени разработки вы тратите на его настройку, оптимизируя его, защищая вещи, блокируя его, масштабируя ресурсы и отслеживая трафик.

Каждый день я вижу, как разработчик борется за защиту своего сервера от какой-то группы хакеров, испытывая проблемы с получением какого-то случайного патча безопасности, пытаясь перейти на более мощные серверы, злится, пытаясь настроить Capistrano, и этот список можно продолжить. Такие сервисы, как Pagoda Box, могут помочь вам пропустить все это; поэтому давайте посмотрим, как это сделать.

Шаг 1 - Регистрация Pagoda Box

Первым шагом является получение учетной записи. Это бесплатно, так что вперед.

Шаг 2 - Создайте приложение

Каждое приложение должно иметь уникальное имя, так как оно также используется для URL-адресов предварительного просмотра. Вы можете назвать свое приложение, как хотите, так как вскоре будете скрывать его за доменом, но назовите его чем-то очевидным и запоминающимся.

Шаг 3 - Настройка Git

Git - чрезвычайно полезная система управления версиями. В настоящее время он широко популярен благодаря таким сайтам как GitHub, которые разрешают бесплатный хостинг, но также отлично подходит для развертывания вашего кода.

Если вы новичок в Git, попробуйте наш курс.

Если вы еще этого не сделали, настройте Git. Их документация поможет вам начать работу. Для пользователи Windows все это будет немного сложнее, чем для пользователей OSX или Linux, поэтому, если у вас будет виртуальная машина Ubuntu, то было бы здорово запустить ее.

Шаг 4 - Создайте свой SSH-ключ

Чтобы запушить ваш код в Pagoda Box, вам необходимо авторизовать свой компьютер для доступа к приложениям в вашем аккаунте. Создайте ключ SSH и вставьте его здесь.

Шаг 5 - Развертывание своего кода

Сначала настройте папку для своего приложения и установите пути Git. Пути - это на самом деле псевдонимы для URL-адреса репозитория. Установив этот путь, мы разрешаем нашему местному реестру Git знать, как пушить код в Pagoda Box.

  $ mkdir my-awesome-app && cd my-awesome-app$ git initInitialized empty Git repository in /Users/phil/Sites/my-awesome-app/.git/$ git remote add pagoda git@git.pagodabox.com:my-awesome-app.git

Теперь, когда все настройки учетной записи и среды выполнены, мы можем начать развертывание кода. На данный момент мы развернем простое приложение Hello World:

$ echo "<?php echo "Hello World!";?>" > index.php

Это небольшой трюк командной строки, чтобы поместить некоторый текст в файл. Вы также можете создать этот файл с помощью вашего любимого текстового редактора или IDE.

$ git add index.php $ git commit -m "OH HAI"[master (root-commit) 6aa23f4] OH HAI 1 files changed, 1 insertions(+), 0 deletions(-) create mode 100644 index.php

До сих пор мы создали новую папку для нашего сайта, инициализировали Git, добавили путь, который сообщает Git, где живет приложение Pagoda Box, и добавил новый файл index.php, который просто выведет Hello World!. Затем:

$ git push pagoda masterCounting objects: 3, done.Writing objects: 100% (3/3), 244 bytes, done.Total 3 (delta 0), reused 0 (delta 0):: Auto deploy enabled for "master" +> to change, visit https://dashboard.pagodabox.com/apps/my-awesome-app/admin:: Deploying to 6aa23f4 on master:: Parsing Boxfile:: Executing deploy +> Init submodules +> Booting web1.1  - [22-Feb-2012 20:38:41] NOTICE: fpm is running, pid 54  - [22-Feb-2012 20:38:41] NOTICE: ready to handle connections +> Propagating network updates:: Cleaning upTo git@git.pagodabox.com:my-awesome-app.git * [new branch]  master -> master

Здесь происходит волшебство. Вы будете выкатывать изменения на удаленный Git, как и любой другой репозиторий, но затем вы увидите, что вывод Pagoda Box захватывает и создает развертывание. Это каждый раз запускает новый экземпляр и мгновенно переключается с реального экземпляра на этот новый экземпляр, что означает немедленные обновления - в отличие от медленных систем развертывания файлов на основе FTP. Кроме того, в отличие от некоторых подобных сервисов, все ваши временные файлы (кеш, журналы, загрузки изображений и т.д.) Будут доступны после любого изменения размера или повторного развертывания. Потрясающие!

Теперь, если вы обновите страницу Pagoda Box, вы увидите панель инструментов со всеми вариантами.

Шаг 6 - Создание Boxfile

В Pagoda Box есть специальный файл конфигурации: Boxfile. Это находится в корне вашего приложения. Расширенный пример может выглядеть примерно так:

web1:  ################################################  ## GENERAL SETTINGS  ################################################  shared_writable_dirs: [/system/cms/cache, /system/cms/logs, /uploads, /addons]  ################################################  ## WEB SPECIFIC SETTINGS  ################################################  index_list: [index.php]  ################################################  ## PHP SPECIFIC SETTINGS  ################################################  php_version: 5.3.8  php_extensions: [mysqli, curl, gd, mbstring]  php_date_timezone: Europe/London  php_max_execution_time: 30  php_max_input_time: 60  php_post_max_size: 25M  php_file_uploads: On  php_upload_max_filesize: 20M  php_max_file_uploads: 20

В этом примере вы можете создать папки для записи, установить свой index_list (который по умолчанию является index.php) и изменить всевозможные настройки PHP, которые обычно находятся в файле php.ini.

Самая важная строка здесь:

php_extensions: [mysqli, curl, gd, mbstring]

Pagoda Box позволяет вам перечислять зависимости приложений. Драйвер mysqli - это «MySQL Improved», который вы должны использовать, поскольку «mysql» будет считаться устаревшим в PHP 5.4. «curl» и «gd» являются довольно стандартными, а «mbstring» помогает вам работать с кодом UTF-8.

Приятно знать, что они доступны - вместе с множеством других, но для этого урока нам потребуется только следующее:

php_extensions: [pdo, pdo_mysql]

Итак, давайте сделаем наш новый Boxfile и сохраним его:

web1:  php_extensions: [pdo, pdo_mysql]

Мы будем использовать это позже.

Шаг 7 - Создание базы данных.

На момент написания этой статьи Pagoda Box поддерживал только базы данных MySQL, и это нормально, так как они используются большинством разработчиков.

Мы можем запускать как можно больше небольших экземпляров базы данных, как нам нравится; мы только начинаем платить, когда они становятся больше, поэтому вы не будете удивлены огромным счетом только за создание веб-сайта. Как и веб-экземпляры, их также можно масштабировать, чтобы расти с вашего сайта.

Чтобы создать новую, щелкните значок + и дайте ей имя. Во время сборки вы увидите индикатор выполнения; затем страница обновится.

Шаг 8 - Подключение из вашего приложения

Подключение через PHP довольно просто. Вам предоставляется несколько переменных $ _SERVER, которые содержат учетные данные для каждой имеющейся у вас базы данных. Это означает, что вы можете хранить жестко запрограммированные пароли из своего приложения, что очень полезно, если вы не хотите, чтобы все ваши разработчики видели живые пароли, или если у вас есть полный сайт на GitHub, как некоторые люди начинают делать ,

$db = new PDO(sprintf("mysql:host=%s;dbname=%s;port=%s", $_SERVER["DB1_HOST"], $_SERVER["DB1_NAME"], $_SERVER["DB1_PORT"]), $_SERVER["DB1_USER"], $_SERVER["DB1_PASS"]);

Это базовый пример используемых переменных - подключение к базе данных MySQL через расширение PDO. Если вы используете фреймворк, такой как CodeIgniter, вы можете перетащить эти переменные в файлы конфигурации database.php.

Шаг 9 - Удаленное подключение

Некоторые веб-серверы, в особенности те, которые установлены под управлением cPanel, устанавливают phpMyAdmin и могут быть настроены на удаленный доступ к базам данных MySQL.

Вместо этого подхода Pagoda Box использует Ruby Gem для - помимо прочего - создания временного безопасного SSH-туннеля. Это означает, что ваши базы данных MySQL заблокированы за брандмауэром и доступны только через это безопасное соединение SSH.

Итак, чтобы установить gem выполним:

$ sudo gem install pagoda

Затем, чтобы создать туннель, просто запустите:

$ pagoda tunnel db1

В первый раз, когда вы запустите это (или любую другую команду pagoda gem), вас попросят ввести имя пользователя и пароль. Это ваше имя пользователя Pagoda Box и пароль, и он необходим, чтобы gem мог действовать от вашего имени и не имеет ничего общего с вашей базой данных.

$ pagoda tunnel db1it appears this is the first time you have used our client  Username: someguy  Password: Tunnel Established! Перейдите в свою панель управления и нажмите на базу данных, затем нажмите «Показать учетные данные», чтобы увидеть информационное окно, подобное этому:

Используйте эти учетные данные для подключения. Не забудьте указать порт; он указан в выводе из gem выше - не зеленый квадрат. Помните, что вы подключаетесь к локальному туннелю, а не напрямую к базе данных.

Шаг 10 - Создание блога

Каким будет учебник, если бы он не включал создание блога через 20 минут?

Для всех разработок вы действительно должны строить вещи локально с помощью MAMP, XAMPP, WAMP и т.д., А затем развертывать изменения кода и базы данных с помощью каких-то «Migrations» или элементов отслеживания изменений схемы, но мы сделаем все по-старому.

Сначала подключитесь к своей базе данных удаленно через туннель, затем запустите этот запрос, чтобы построить новую таблицу:

Шаг 11 - Использование реального домена

Нет смысла иметь ваш замечательный новый блог на субдомене pagodabox.com; поэтому давайте разместим домен поверх него.

Во-первых, на вкладке DNS/SSL вашей информационной панели добавьте некоторые записи для своего домена:

Теперь, когда ваше приложение готово принять домен, вы должны руководить тем, кто обрабатывает DNS для вашего домена. Перейдите в область управления DNS и установите для A записи «myawesomedomain.com» запись «50.97.141.37» - это IP-адрес, указанный в интерфейсе для вашего приложения. Эти IP-адреса не уникальны для каждого приложения, но не всегда будут одинаковыми. Установите запись CNAME для «www.myawesomedomain.com» на псевдоним «myawesomedomain.com», и все должно быть настроено.

Заключение

Все это может показаться немного отличным от того, как вы работаете в данный момент, но начинайте думать о том, сколько работы вы спасете времени в долгосрочной перспективе. Использование PaaS - та же самая причина, по которой мы используем jQuery над нативным JavaScript или фреймворками PHP над написанием собственного PHP. Зачем вмешиваться в мелкие детали, когда вы можете сосредоточиться на создании потрясающего приложения?

Другие альтернативы, которые вы можете рассмотреть, это PHPFog и Orchestr.io, которые предлагают подобные услуги. Получайте удовольствие и спасибо за чтение!
