Создание веб-приложения Node с интеграцией Hapi и Twilio...

Code Web Development
Tutorial

Будучи евангелистом-разработчиком Twilio, я провожу много времени на мероприятиях. Когда я общаюсь с организаторами мероприятий, и они понимают, что я работаю для Twilio, один из самых распространенных запросов, которые у них есть, для меня - простое приложение, которое они могут использовать для голосования на своих мероприятиях. Как правило, они пытаются получить обратную связь от своих участников, а SMS - это единственное приложение, которое все установили.

Их требования, как правило, включают в себя возможность голосовать с использованием SMS, не позволяя людям голосовать более одного раза, а иногда и показывать результаты голосования в режиме реального времени. Со временем я создал несколько итераций этого приложения и немного узнал о создании веб-приложений с помощью Node и Angular.

Введение

На протяжении этого учебного пособия в трех частях мы собираемся с нуля создать полноценное сетевое приложение Node/Angular. Вместо создания упрощенного приложения, мы собираемся создать реальную часть программного обеспечения, которая интегрируется с сторонним API (Twilio), сохраняет информацию в базе данных (CouchDB), отображает информацию в режиме реального времени (Socket.io ) и многое, многое другое. В первой части этого урока я собираюсь провести вас через:

  1. Создание приложения Node
  2. Добавление веб-фрейсвокра Hapi
  3. Интеграция с Twilio

Подготовка

В этом учебнике предполагается, что у вас есть доступ к терминалу на вашей машине разработки. Вам также нужно будет сделать следующее:

  • Установить Node
  • Зарегистрировать себе бесплатную учетную запись разработчика Twilio
  • Установите Ngrok (необязательно)

Начало работы с Node & Hapi

Теперь, когда у вас установлен Node, создание нового приложения Node будет довольно просто. Сначала откройте терминал. Создайте новый каталог для своего приложения, перейдите в этот каталог и запустите:

npm init

Это проведет вас через процесс создания файла package.json, который определяет основы вашего приложения (имя, зависимости и т.д.). Не стесняйтесь нажимать кнопку ввода в каждом приглашении, чтобы придерживаться значений по умолчанию. Затем, давайте установим веб-фреймворк Hapi:

npm install hapi --save

Я выбрал Hapi над более популярной платформой Express по двум причинам:

  1. Он поставляется предварительно настроенным для обработки основных задач веб-приложений (например, обработки HTTP POST).
  2. Он также сосредоточен на создании приложений и услуг.

Создайте новый файл index.js и откройте его в своем редакторе кода. Давайте рассмотрим несколько простых строк кода, чтобы запустить наше приложение:

var Hapi = require("hapi");var server = Hapi.createServer("localhost", process.env.PORT || 3000);

Этот код делает модуль Hapi доступным в нашем коде, создает сервер, привязанный к localhost, и делает веб-приложение доступным в любом порту, определяемом process.env.PORT или 3000, если эта переменная среды не определена. Этот код необходим, чтобы при развертывании этого приложения в боевой среде он привязывался к соответствующему порту. Затем определим маршрут:

server.route([{  method: "GET",  path: "/vote/sms",  handler: function (request, reply) {    reply("hello world");  }}]);

Маршрут - это всего лишь кортеж, состоящий из HTTP-метода, пути URL и функции обработчика. Итак, мы говорим Hapi, что если какие-либо запросы GET приходят на путь /vote/sms, отправьте ответ со строкой "hello world". Теперь добавьте эту окончательную строку:

server.start();

Это запустит сервер на вашей машине разработки, который прослушивает порт 3000. Сохраните index.js и вернитесь к своему терминалу. Запустите приложение следующим образом:

node .

Теперь наше приложение работает. Откройте браузер и перейдите по адресу http://localhost:3000/vote/sms. Вы должны увидеть следующее:

Это мои друзья, это Hello World, используя Node и Hapi.

Обработка SMS с помощью Twilio

Теперь, когда у нас есть функциональное приложение Node, мы можем начать думать об обработке голосов с помощью SMS. Для этого мы собираемся интегрировать наше приложение Node с Twilio, сторонним API, который помогает разработчикам отправлять и получать текстовые сообщения, совершать и принимать телефонные звонки и вставлять VOIP в их веб-приложение.

В нашем случае мы хотим обрабатывать входящие текстовые сообщения. Twilio использует конструкцию, известную как webhook, для уведомления веб-приложений о входящих текстовых сообщениях или телефонных звонках. Webhook - это просто URL-адрес, настроенный разработчиком, который вызывается в ответ на событие. Вот простая схема того, как Twilio webhooks будет работать с нашим приложением Node:

Когда Twilio делает HTTP-запрос (либо GET, либо POST, также настраиваемый разработчиком), он также будет включать в себя четко определенный набор параметров, включая информацию, такую как текстовое сообщение, тело текстового сообщения и так далее. Когда ваше веб-приложение обработает запрос, Twilio ожидает ответа в виде правильно отформатированного документа TwiML. Этот XML-документ предоставляет дополнительные инструкции для Twilio для выполнения.

Чтобы более легко работать с Twilio API, давайте установим модуль Twilio NPM. Откройте терминал, зайдите в корневую директорию своего приложения и выполните следующую команду:

npm install twilio --save

Создание маршрута для нашего веб-узла

Теперь, когда у нас есть понимание на высоком уровне того, как работает Twilio, давайте создадим URL-адрес, который Twilio может вызвать, когда приходит голосование по SMS. В своем каталоге приложений создайте подкаталог, называемый routes. В этом новом каталоге создайте новый файл с именем index.js. В этом файле будет размещена логика маршрутизации URL-адреса нашего приложения, чтобы не загромождать наш основной файл приложения. Вы увидите этот шаблон снова и снова, поскольку мы добавим больше сложности в наше приложение. Сохранение модульности позволяет легче понять и протестировать наше приложение. Теперь добавьте следующий код:

var twilio = require("twilio")  , Hapi;module.exports = function(hapi) {  Hapi = hapi  return exports;};var voteSMS = exports.voteSMS = function(request, reply) {  var resp = new twilio.TwimlResponse();  resp.message("Thanks for voting!");  reply(resp.toString()).type("text/xml");};

Здесь происходит три вещи:

  1. Нам подключаем модуль Twilio Node
  2. Мы переопределяем module.exports, чтобы принять ссылку на Hapi
  3. Мы определяем обработчик маршрута

Обработчик маршрута немного отличается от первого, который мы написали. На этой итерации мы используем модуль Twilio для создания и заполнения объекта TwimlResponse. Затем мы преобразуем этот объект в String и передадим его методу ответа. Наконец, мы явно устанавливаем Content-Type в text/xml. Это станет привычным образцом, когда мы будем отвечать на запросы Twilio webhook.

Чтобы завершить этот пример, откройте свой основной файл index.js (в корневой каталог вашего приложения) и добавьте следующую строку кода ниже, где вам нужен модуль Hapi:

var routes = require("./routes")(Hapi);

Затем отредактируйте маршрут, который вы определили, чтобы удалить анонимную функцию и замените его ссылкой на routes.voteSMS. Теперь перезапустите сервер node и зайдите в http://localhost:3000/vote/sms в своем браузере. Вы увидите:

Настройка вашего веб-хука

Теперь, когда у нас есть работающий маршрут, следующий шаг - настроить Twilio на запрос этого URL-адреса всякий раз, когда приходит SMS. Чтобы это произошло, URL-адрес, который мы предоставляем Twilio, должен быть общедоступным. Настройка Twilio для запроса на localhost не будет работать. В реальном мире ваше приложение будет развернуто на VPS/хосте по вашему выбору. Тем не менее, было бы здорово, если бы мы смогли удержать развертывание в целях быстрого итеративного развития. Как оказалось, есть замечательный инструмент для этого: Ngrok.

Ngrok - небольшая часть программного обеспечения, которая создает безопасный туннель между вашей локальной машиной и общедоступным доменом. Он полезен для тестирования приложений Twilio, но вы также можете использовать его для обмена живыми URL-адресами с веб-приложениями, запущенными на вашей машине разработки, для целей тестирования, обратной связи или других задач.

Предполагая, что вы установили Ngrok, убедитесь, что ваш Node-сервер запущен, а затем, в другом терминале, запустите Ngrok:

ngrok 3000

Вы должны увидеть такой экран:

Вы можете подтвердить, что ваше локальное веб-приложение Node действительно работает в общедоступном Интернете, скопировав следующий URL-адрес в ваш браузер: https://YOUR-SUBDOMAIN.ngrok.com/vote/sms. Если вы видите ранее знакомый XML, то можно двигаться дальше!

Теперь мы можем настроить Twilio. Войдите в свою учетную запись Twilio, перейдите на вкладку «Numbers» и выберите номер для работы. Если вы создали бесплатную учетную запись разработчика, вам должно быть присвоено число Twilio. В разделе «Messaging» установите URL-адрес запроса на URL-адрес Ngrok, включая путь /vote/sms:

Обеспечение безопасности Twilio Webhook

Самые внимательные среди вас, возможно, уже заметили небольшую проблему безопасности с нашим приложением. Что нужно, чтобы остановить злоумышленника от определения URL-адреса вашего веб-узла и затем подражать Twilio, отправив свои собственные HTTP-запросы? Это похоже на довольно простой способ заполнить избирательную урну для нашего голосования. К счастью, Twilio ожидал необходимости защищать веб-узлы, а модуль Twilio Node предоставляет метод для проверки того, что HTTP-запрос действительно поступает из Twilio.

Сначала вам понадобится ваш токен Auth. Каждая учетная запись Twilio имеет идентификатор SID учетной записи и токен аутентификации. Они похожи на имя пользователя и пароль, но для доступа к вашей учетной записи API для программных API. Если вы вошли на веб-сайт Twilio, вы найдете обе эти части информации на панели управления:

Нажмите на значок замка, чтобы открыть токен авторизации. Затем выберите его и скопируйте в буфер обмена. Затем создайте файл в каталоге app с именем config.js. Добавьте следующие строки:

var config = {};config.twilio = {};config.twilio.key = "yyy";config.twilio.disableSigCheck = false;module.exports = config;

Теперь откройте routes.js и импортируйте наш новый конфигурационный модуль, добавив следующую строку в начало файла:

var config = require("../config");

Далее, давайте сделаем небольшую модификацию нашего маршрута voteSMS:

var voteSMS = exports.voteSMS = function(request, reply) {  if (fromTwilio(request) || config.twilio.disableSigCheck) {    var resp = new twilio.TwimlResponse();    resp.message("Thanks for voting!");    reply(resp.toString()).type("text/xml");  }  else {    reply(Hapi.error.unauthorized("Sorry, no black hats allowed"));  }};

Все, что мы делаем, это вызов функции, чтобы проверить, действительно ли это запрос от Twilio. Если да, тогда продолжайте. Если это не так, верните 403 Unauthorized ошибку клиенту. Функция fromTwilio довольно проста:

var fromTwilio = function(request) {  var sig  = request.headers["x-twilio-signature"]    , url  = config.twilio.messagingUrl + request.url.search    , body = request.payload || {};  return twilio.validateRequest(config.twilio.authToken, sig, url, body);};

Здесь мы просто используем метод validateRequest модуля Twilio Node, чтобы узнать, пришел ли запрос от Twilio. Те, кто заинтересован, должны больше узнать о его реализации, но давайте двигаться дальше. Функция принимает четыре параметра: ваш Twilio Auth Token, значение заголовка, переданное Twilio, полный URL-адрес, который Twilio вызывал (включая любые параметры GET) и тело POST (пустое, если это не был запрос POST). Он вернет true, если запрос поступит от Twilio и false в противном случае.

Это легко проверить, если это работает. Во-первых, отправьте SMS-сообщение на номер Twilio и посмотрите, получаете ли вы ответ. Затем откройте свой браузер и попытайтесь открыть URL-адрес. Вы должны увидеть следующее:

Защищено!
