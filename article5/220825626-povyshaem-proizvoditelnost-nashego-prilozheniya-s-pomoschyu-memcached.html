Повышаем производительность нашего приложения с помощью Memcached...

Code Web Development
Tutorial

Одним из самых простых способов повысить производительность вашего приложения является создание кэширующего слоя перед вашей базой данных. В этом уроке я покажу вам, как использовать Memcached с Rails, Django или Drupal.

Memcached - отличный выбор для этой проблемы, учитывая его солидную историю, простую установку и активное сообщество. Он используется многими известными компаниями включая таких гигантов, как Facebook, YouTube и Twitter. Сайт Memcached сам по себе хорошо описывает Memcached как «бесплатную и открытую, высокопроизводительную систему кэширования объектов с распределенной памятью, универсальную по своей природе, но предназначенную для ускорения динамических веб-приложений путем облегчения загрузки базы данных. "

Часто запросы к базе данных медленные.

В общем случае вызовы базы данных выполняются медленно, поскольку запрос для обработки требует ресурсы процессора, а сами данные (обычно) извлекаются с диска. С другой стороны, кеш в памяти, такой как Memcached, занимает очень мало ресурсов ЦП, а данные извлекаются сразу из памяти вместо диска. Отсутствие затрат на процессор является эффектом дизайна Memcached; он не поддерживает запросы, как база данных SQL. Вместо этого он использует пары «ключ-значение» для извлечения всех данных, и вы не можете извлечь данные из Memcached, не зная ключ.

Memcached хранит пары ключ-значение целиком в памяти. Это делает поиск чрезвычайно быстрым, но также делает его таким, чтобы данные были эфемерными. В случае сбоя или перезагрузки память очищается, и все пары ключ-значение необходимо перестроить. В Memcached нет встроенных высокодоступных и / или отказоустойчивых механизмов. Однако это распределенная система, поэтому данные хранятся на нескольких узлах. Если один узел потерян, остальные узлы выполняют запросы и заполняют недостающий узел.

Установка Memcached

Установка Memcached представляет собой довольно простой процесс. Ее можно сделать с помощью диспетчера пакетов или путем компиляции из исходников. В зависимости от вашего дистрибутива вы можете захотеть скомпилировать исходники, поскольку пакеты, как правило, немного отстают в версиях.

# Install on Debian and Ubuntuapt-get install memcached# Install on Redhat and Fedorayum install memcached# Install on Mac OS X (with Homebrew)brew install memcached# Install from Sourceget http://memcached.org/latesttar -zxvf memcached-1.x.x.tar.gzcd memcached-1.x.x./configuremake && make testsudo make install

Возможно вы захотите настроить Memcached для ваших конкретных потребностей, но в этом примере мы просто запустим его с некоторыми базовыми настройками.

memcached -m 512 -c 1024 -p 11211 -d

На этом этапе Memcached уже должен быть запушен. Затем мы рассмотрим, как использовать его с Rails, Django и Drupal. Следует отметить, что Memcached не ограничивается использованием в рамках какого-либо фреймворка. Вы можете использовать Memcached со многими языками программирования и с любым из многочисленных доступных клиентов.

Использование Memcached с Rails 3

Rails 3 абстрагировал систему кэширования, чтобы вы могли свободно изменять клиента по своему желанию. В Ruby предпочтительным клиентом Memcached является Dalli.

# Add Dalli to your Gemfilegem "dalli"# Enable Dalli in config/environments/production.rb:config.perform_caching = trueconfig.cache_store = :dalli_store, "localhost:11211"

В режиме разработки вы обычно не используете Memcached, поэтому либо запускайте Rails в боевом режиме комадной rails server -e production, либо добавьте вышеуказанные строки в config/environments/development.rb.

Простейшее использование кеша - это методы write/read для извлечения данных:

Rails.cache.write "hello", "world"      #=> trueRails.cache.read "hello"                #=> "world"

Наиболее распространенным шаблоном для кеширования Rails является использование fetch. Он попытается извлечь ключ (в данном случае expensive-query) и вернуть значение. Если ключ не существует, он выполнит переданный блок и сохранит результат по данному ключу.

Rails.cache.fetch "expensive-query" do  results = Transaction.    joins(:payment_profile).    joins(:order).    where(":created > orders.created_at", :created => Time.now)end# ... more code working with results

В приведенном выше примере проблема заключается в истечении срока действия кеша. (Одна из двух трудных проблем в информатике.) Продвинутым, очень надежным решением является использование некоторой части результатов в самом кеше, поэтому, если результаты будут изменены, ключ истечет автоматически.

users = User.activeusers.each do |u|  Rails.cache.fetch "profile/#{u.id}/#{u.updated_at.to_i}" do    u.profile  endend

Здесь мы используем эпоху update_at как часть ключа, которая дает время истечения ключа. Итак, если время user.updated_at изменится, мы получим пустой кэш в уже существующем кэше профиля и запишем новый. В этом случае нам нужно будет обновить время update_at пользователя, когда будет обновлен профиль. Это так же просто, как добавление:

class Profile < ActiveRecord::Base  belongs_to :user, touch: trueend

Теперь у вас есть профили с автоматическим изменением срока действия кеша без каких-либо проблем с получением старых данных, когда пользователь обновляется. Это похоже на волшебство!

Использование Memcached с Django

После установки Memcached доступ к нему из Django будет довольно прост. Во-первых, вам нужно будет установить клиентскую библиотеку. Мы будем использовать pylibmc.

# Install the pylibmc librarypip install pylibmc# Configure cache servers and binding settings.pyCACHES = {    "default": {        "BACKEND": "django.core.cache.backends.memcached.PyLibMCCache",        "LOCATION": "127.0.0.1:11211",    }}

Теперь ваше приложение должно работать с Memcached. Как и в других библиотеках, вы получите базовые getter и setter для доступа к кешу:

cache.set("hello", "world")cache.get("hello")             #=> "world"

Вы можете условно установить ключ, если он еще не существует с помощью add. Если ключ уже существует, новое значение будет проигнорировано.

cache.set("hello", "world")cache.add("hello", "mundus")cache.get("hello")              #=> "world"

Из библиотеки Python Decorator Library вы можете создать memoized декоратор для кэширования результатов вызова метода.

import collectionsimport functoolsclass memoized(object):    """Decorator. Caches a function"s return value each time it is called.    If called later with the same arguments, the cached value is returned    (not reevaluated).    """    def __init__(self, func):        self.func = func        self.cache = {}    def __call__(self, *args):        if not isinstance(args, collections.Hashable):            # uncacheable. a list, for instance.            # better to not cache than blow up.            return self.func(*args)        if args in self.cache:            return self.cache[args]        else:            value = self.func(*args)            self.cache[args] = value            return value    def __repr__(self):        """Return the function"s docstring."""        return self.func.__doc__    def __get__(self, obj, objtype):        """Support instance methods."""        return functools.partial(self.__call__, obj)@memoizeddef fibonacci(n):    "Return the nth fibonacci number."    if n in (0, 1):        return n    return fibonacci(n-1) + fibonacci(n-2)print fibonacci(12)

Декораторы могут дать вам возможность взять большую часть тяжелого подъема из кеширования и истечения срока действия кеша. Обязательно ознакомьтесь с примерами кэширования в библиотеке Decorator, когда будете планировать свою систему кэширования.

Использование Memcached в Drupal

Начало работы с Memcached в Drupal начинается с установки расширения PHP для Memcached.

# Install the Memcached extensionpecl install memcache<?php    // Configure Memcached in php.ini    [memcache]    memcache.hash_strategy = consistent    memcache.default_port = 11211?><?php    // Tell Drupal about Memcached in settings.php    $conf["cache_backends"][] = "sites/all/modules/contrib/memcache/memcache.inc";    $conf["cache_default_class"] = "MemCacheDrupal";    $conf["memcache_key_prefix"] = "app_name";    $conf["memcache_servers"] = array(        "10.1.1.1:11211" => "default",        "10.1.1.2:11212" => "default"    );?>

Вам нужно будет перезапустить приложение, чтобы все изменения вступили в силу.

Как и ожидалось, вы получите стандартные getter и setter с модулем Memcached. Одно из предостережений заключается в том, что cache_get возвращает строку кэша, поэтому вам нужно получить доступ к сериализованным данным внутри нее.

<?php    cache_set("hello", "world");    $cache = cache_get("hello");    $value = $cache->data;  #=> returns "world"?>

И точно так же у вас есть кэширование в Drupal. Вы можете создавать пользовательские функции для репликации таких функций, как cache.fetch в Rails. Немного проектирования и вы можете получить надежное решение для кеширования, которое перенесет отзывчивость вашего приложения на новый уровень.

Мы это сделали

В то время как хорошая стратегия кэширования требует времени, она не должна мешать вам начать работу.

Реализация системы кеширования может быть довольно простой. При правильной конфигурации решение для кеширования может продлить срок службы вашей текущей архитектуры и сделать ваше приложение более приятным, чем когда-либо раньше. В то время как хорошая стратегия кэширования требует времени для уточнения, это не должно мешать вам начать работу.

Как и в любой сложной системе, мониторинг имеет решающее значение. Понимание того, как используется ваш кеш, и где находятся горячие точки ваших данных, поможет вам улучшить производительность кэша. Memcached имеет систему статистики качества, которая поможет вам контролировать кластер кэша. Вы также должны использовать инструмент, такой как New Relic, чтобы следить за балансом между кешем и базой данных. В качестве дополнительного бонуса вы можете получить бесплатную футболку «Data Nerd», когда зарегистрируетесь.
