Шаблоны проектирования: Шаблон Команда...

Code Design Patterns
Tutorial

После рассмотрения нескольких порождающих и структурных шаблонов проектирования, мы переходим к поведенческим шаблонам. Это первая статья в этой категории шаблонов. Порождающие шаблоны показали, как нам следует создавать наши объекты, структурные шаблоны демонстрируют, каким образом следует структурировать классы и объекты в приложении.

В этой статье мы изучим шаблон проектирования команда. Как видно из названия, этот шаблон будет иметь дело с выполнением различных команд. Вот что говорит нам об этом шаблоне Wikipedia:

Это поведенческий шаблон проектирования, используемый при объектно-ориентированном программировании, представляющий действие. Объект команды заключает в себе само действие и его параметры. Эта информация включает в себя имя метода, объект которому принадлежит метод и значения входных параметров.

Обычно этот шаблон состоит из множества элементов. В следующем разделе мы подробно рассмотрим каждый элемент отдельно вместе с примерами кода. Я буду рассматривать примеры с действиями над радио - самыми базовыми действиями могут быть включение и выключение радио. Итак, давайте погрузимся в каждый элемент.

Приемник

Этот элемент содержит фактическую реализацию любой команды (знает как выполнять запрошенные команды). Он так же поддерживает историю выполненных команд, но это не является частью шаблона. Мы увидим эту часть в шаблоне проектирования Хранитель.

// Receiverclass radioControl {    public function turnOn() {  // Turning On Radioecho "Turning On Radio";}public function turnOff() {// Turning Off Radioecho "Turning Off Radio";}}

Команда

Этот элемент содержит информацию о действии, которое следует выполнить. Он вызывает необходимый метод у receiver.

// Commandinterface radioCommand {    public function execute();}class turnOnRadio implements radioCommand {private $radioControl;public function __construct(radioControl $radioControl) {$this->radioControl = $radioControl;}public function execute() {$this->radioControl->turnOn ();}}class turnOffRadio implements radioCommand {private $radioControl;public function __construct(radioControl $radioControl) {$this->radioControl = $radioControl;}public function execute() {$this->radioControl->turnOff ();}}

Клиент

Этот элемент действительно ведет себя как клиент (решает что следует выполнить). Его задача - определить какую команду следует выполнить, ему не важно кто и как будет ее исполнять. В этом примере и взял команду как жестко прописанное значение, но она может быть получена откуда угодно, например из URL или из переменной.

// Client$in = "turnOffRadio";

Вызыватель

Элемент который инициализирует весь процесс. Он принимает аргументы от клиента и запускает процесс для вызова необходимой команды.

// Invokerif (class_exists ( $in )) {    $command = new $in ( new radioControl () );} else {throw new Exception ( "..Command Not Found.." );}$command->execute ();

Как только мы выполним эту команду, будет показано сообщение "Turning Off Radio", потому что клиент применил эту команду. Дополнительные команды могут быть выполнены таким же способом. Если бы наше действие было установлено как turnOffRadio, то было бы показано сообщение "Turning On Radio". Достаточно просто, не так ли?

Собираем Все Вместе

Давайте соберем все элементы вместе, чтобы увидеть как это работает.

// Receiverclass radioControl {    public function turnOn() {// Turning On Radioecho "Turning On Radio";}public function turnOff() {// Turning Off Radioecho "Turning Off Radio";}}// Commandinterface radioCommand {public function execute();}class turnOnRadio implements radioCommand {private $radioControl;public function __construct(radioControl $radioControl) {$this->radioControl = $radioControl;}public function execute() {$this->radioControl->turnOn ();}}class turnOffRadio implements radioCommand {private $radioControl;public function __construct(radioControl $radioControl) {$this->radioControl = $radioControl;}public function execute() {$this->radioControl->turnOff ();}}// Client$in = "turnOffRadio";// Invokerif (class_exists ( $in )) {$command = new $in ( new radioControl () );} else {throw new Exception ( "..Command Not Found.." );}$command->execute ();

Добавление Новых Команд

Ранее мы имели только одну частоту для проигрывания на радио, и у нас было всего два метода: включить и выключить радио. Но представим, что у нас появилось несколько частот на радио, которые требуют чтобы мы добавили два новых метода: tuneUp и tuneDown. Посмотрим, как мы может их добавить.

Когда мы используем шаблоны проектирования, они помогают нам легко вносить изменения или расширять функционал без каких либо изменений в клиентском коде. Это применимо и в нашем случае. Мы собираемся добавить еще две команды, но наш клиент и вызывать останутся при этом без изменений.

Добавление новых команд требует изменений в двух местах. Первое - создать новые команды, которые реализуют интерфейс radioCommand и затем определить фактические реализации этих команд на приемнике.

Новые Команды

class tuneUpRadio implements radioCommand {    private $radioControl;public function __construct(radioControl $radioControl) {$this->radioControl = $radioControl;}public function execute() {$this->radioControl->tuneUp ();}}class tuneDownRadio implements radioCommand {private $radioControl;public function __construct(radioControl $radioControl) {$this->radioControl = $radioControl;}public function execute() {$this->radioControl->tuneDown ();}}

Обновленный Код Приемника

Мы будем добавлять два новых метода в наш класс приемника.

// Receiverclass radioControl {    public function turnOn() {// Turning On Radioecho "Turning On Radio";}public function turnOff() {// Turning Off Radioecho "Turning Off Radio";}public function tuneUp() {// Tuning Up Radioecho "Tuning Up Radio";}public function tuneDown() {// Tuning Down Radioecho "Tuning Down Radio";}}

Заключение

Итак, нам следует использовать шаблон проектирования Команда, когда есть необходимость в выполнении нескольких команд, и не важно связаны ли команды между собой. Я старался изо всех сил, чтобы подробно остановиться на этом шаблоне проектирования, и вы можете оставить свой отзыв в комментариях.
